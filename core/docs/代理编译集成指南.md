# EIPPF 代理编译集成指南（中文）

本文档说明如何把本项目的代理编译能力无缝接入到常见开发环境，达到“像正常编译一样点击 Build，但产物自动完成保护”的目标。

适用范围：
- C/C++（Clang/GCC/MSVC/clang-cl）
- Rust（rustc/cargo，含透明链接）
- Android NDK（CMake 与 ndk-build）
- 任意支持“编译器启动器/包装器”的 IDE/构建系统

---

## 1. 核心概念

EIPPF 通过 `core/wrapper/weaver_proxy.py` 作为“编译器代理”工作：

1. 编译阶段：源码 -> LLVM Bitcode  
2. 保护阶段：`ip_weaver_ir` 对 Bitcode 做 IR 级保护改写  
3. 回落阶段：将改写后的 Bitcode 继续编译为目标文件  
4. 链接阶段：自动注入 VM Runtime（`libeippf_vm_rt.a/.lib`）与必要链接参数

对开发者来说，调用方式保持“编译器原命令行语义”，只是通过 launcher 包一层。

---

## 2. 前置准备

先在本仓库构建核心组件：

```bash
cmake -S core -B build -G Ninja
cmake --build build -j --target ip_weaver_ir eippf_vm_rt
```

关键路径：
- 代理脚本：`core/wrapper/weaver_proxy.py`
- IR Weaver：`build/ip_weaver_ir/ip_weaver_ir`
- VM Runtime：`build/runtime/libeippf_vm_rt.a`（Windows 下为 `.lib`）

要求：
- Python3 可用
- LLVM/Clang 工具链可用
- Rust 场景需 `rustc/cargo`

---

## 3. CMake 全局零侵入接入（推荐）

本项目已提供 `core/cmake/WeaverLauncher.cmake`。  
在你的顶层 `CMakeLists.txt` 中：

```cmake
include("${CMAKE_SOURCE_DIR}/core/cmake/WeaverLauncher.cmake")

eippf_enable_weaver_launcher(
  PROXY_SCRIPT "${CMAKE_SOURCE_DIR}/core/wrapper/weaver_proxy.py"
  IR_WEAVER_BIN "${CMAKE_BINARY_DIR}/ip_weaver_ir/ip_weaver_ir"
  VM_RUNTIME_LIB eippf_vm_rt
)
```

效果：
- 自动设置 `CMAKE_C_COMPILER_LAUNCHER`
- 自动设置 `CMAKE_CXX_COMPILER_LAUNCHER`
- CMake 3.27+ 下还会设置 `C/CXX_LINKER_LAUNCHER`

开发者后续只需正常：
- `cmake --build ...`
- IDE 点击 Build

---

## 4. Visual Studio 一键编译（CMake 项目）

建议使用 VS 的 CMake 工作流（不是手写 `.vcxproj`）：

1. 打开项目根目录（含 `CMakeLists.txt`）  
2. 在 `CMakePresets.json` 或 CMake Settings 中设置：
   - 生成器（Ninja/Ninja Multi-Config）
   - `CMAKE_TOOLCHAIN_FILE`（如有）
3. 在 CMake 配置阶段调用 `eippf_enable_weaver_launcher(...)`

示例 `CMakePresets.json` 片段：

```json
{
  "version": 3,
  "configurePresets": [
    {
      "name": "vs-release",
      "generator": "Ninja",
      "binaryDir": "${sourceDir}/build-vs",
      "cacheVariables": {
        "CMAKE_BUILD_TYPE": "Release"
      }
    }
  ]
}
```

注意：
- VS 里点击 Build 本质是调用 CMake 构建；launcher 配置生效后即自动走代理。
- Windows 下若使用 `cl.exe/clang-cl`，代理会自动区分 MSVC 风格参数。

## 5. Clang/GCC 命令行接入（与原命令一致）

### 5.1 C/C++ 示例

```bash
python3 core/wrapper/weaver_proxy.py \
  --ir-weaver-bin build/ip_weaver_ir/ip_weaver_ir \
  --vm-runtime-lib build/runtime/libeippf_vm_rt.a \
  clang -O2 -c app.c -o app.o

python3 core/wrapper/weaver_proxy.py \
  --ir-weaver-bin build/ip_weaver_ir/ip_weaver_ir \
  --vm-runtime-lib build/runtime/libeippf_vm_rt.a \
  clang app.o -o app
```

说明：  
非编译指令（如纯链接）代理会透明透传，并自动注入 runtime 链接参数。

## 6. Android NDK 接入（CMake + ndk-build）

### 6.1 CMake / ExternalNativeBuild（推荐）

推荐走 CMake（Android Studio/Gradle ExternalNativeBuild）：

1. 正常使用 NDK toolchain（`android.toolchain.cmake`）  
2. 继续设置 `CMAKE_C_COMPILER_LAUNCHER/CMAKE_CXX_COMPILER_LAUNCHER` 为代理  
3. 按 ABI 分别提供对应的 VM Runtime 静态库（`arm64-v8a` / `armeabi-v7a` / `x86_64`）

关键点：
- 代理会保留 `-target/--target` 等跨编译参数，不破坏 NDK 目标三元组。
- VM Runtime 必须和目标 ABI 一致，否则链接失败。

### 6.2 ndk-build 方法一：`NDK_CCACHE`（最稳妥）

该方式不改源码、不改 `Android.mk` 语义，适合快速接入。

Linux/macOS 包装脚本示例（`core/wrapper/ndk_weaver_ccache.sh`）：

```bash
#!/usr/bin/env bash
exec python3 /abs/path/to/core/wrapper/weaver_proxy.py \
  --ir-weaver-bin /abs/path/to/build/ip_weaver_ir/ip_weaver_ir \
  --vm-runtime-lib /abs/path/to/build/runtime/libeippf_vm_rt.a \
  "$@"
```

调用：

```bash
chmod +x core/wrapper/ndk_weaver_ccache.sh
ndk-build NDK_CCACHE=/abs/path/to/core/wrapper/ndk_weaver_ccache.sh
```

Windows 包装脚本示例（`core\\wrapper\\ndk_weaver_ccache.cmd`）：

```bat
@echo off
setlocal
py -3 C:\path\to\core\wrapper\weaver_proxy.py ^
  --ir-weaver-bin C:\path\to\build\ip_weaver_ir\ip_weaver_ir.exe ^
  --vm-runtime-lib C:\path\to\build\runtime\eippf_vm_rt.lib ^
  %*
exit /b %ERRORLEVEL%
```

调用（`cmd.exe`）：

```bat
ndk-build.cmd NDK_CCACHE=C:\path\to\core\wrapper\ndk_weaver_ccache.cmd
```

### 6.3 ndk-build 方法二：编译器前置代理（第二种方法）

该方式将代理伪装为 `clang/clang++`（或目标三元组 clang 驱动），使 ndk-build 全流程都经过代理。  
适用于专用 CI 镜像；本地研发更建议用 6.2。

实施要点：

1. 在独立目录放置“同名编译器包装器”（如 `clang`、`clang++`、`aarch64-linux-android21-clang`）。  
2. 包装器内部调用 `weaver_proxy.py`，并把真实编译器路径通过环境变量传入（例如 `EIPPF_REAL_CC`）。  
3. 让 ndk-build 优先使用该目录中的编译器（通过 toolchain 路径优先级或专用工具链目录）。

Linux/macOS 包装器示例（以 `clang` 为例）：

```bash
#!/usr/bin/env bash
REAL_CC="/opt/android-ndk/toolchains/llvm/prebuilt/linux-x86_64/bin/clang"
exec python3 /abs/path/to/core/wrapper/weaver_proxy.py \
  --ir-weaver-bin /abs/path/to/build/ip_weaver_ir/ip_weaver_ir \
  --vm-runtime-lib /abs/path/to/build/runtime/libeippf_vm_rt.a \
  "$REAL_CC" "$@"
```

Windows 包装器示例（`clang.cmd`）：

```bat
@echo off
setlocal
set REAL_CC=C:\Android\ndk\toolchains\llvm\prebuilt\windows-x86_64\bin\clang.exe
py -3 C:\path\to\core\wrapper\weaver_proxy.py ^
  --ir-weaver-bin C:\path\to\build\ip_weaver_ir\ip_weaver_ir.exe ^
  --vm-runtime-lib C:\path\to\build\runtime\eippf_vm_rt.lib ^
  "%REAL_CC%" %*
exit /b %ERRORLEVEL%
```

说明：
- 第二种方法在 Windows 也可用，但必须使用 `.cmd/.exe` 包装器，不能直接用 `.sh`。  
- 需要为实际参与构建的编译器驱动都提供包装器（含目标三元组 clang 驱动）。

## 7. Rust（rustc/cargo）零改造接入

### 7.1 rustc 直接编译

```bash
python3 core/wrapper/weaver_proxy.py \
  --ir-weaver-bin build/ip_weaver_ir/ip_weaver_ir \
  --vm-runtime-lib build/runtime/libeippf_vm_rt.a \
  rustc main.rs -O -o app
```

### 7.2 cargo 透明接入（推荐）

```bash
RUSTC_WRAPPER=core/wrapper/weaver_proxy.py \
EIPPF_IR_WEAVER_BIN=build/ip_weaver_ir/ip_weaver_ir \
EIPPF_VM_RUNTIME_LIB=build/runtime/libeippf_vm_rt.a \
cargo build
```

说明：
- 已支持 rustc/cargo 链接阶段透明注入 VM Runtime（无需改 `Cargo.toml`）。
- 默认开启 Rust 全覆盖保护（按当前 crate 符号范围），可通过：
  - `EIPPF_RUST_FULL_COVERAGE=0` 暂时关闭。

## 8. 任意 IDE 适配通用方法

核心原则：  
只要 IDE/构建系统支持“Compiler Launcher / Wrapper / Toolchain Executable”，就能无缝接入。

通用配置要素：

1. 启动器命令：
   - `python3 <repo>/core/wrapper/weaver_proxy.py`
2. 必需参数或环境变量：
   - `--ir-weaver-bin <ip_weaver_ir绝对路径>`
   - `--vm-runtime-lib <eippf_vm_rt绝对路径>`
   - 或使用环境变量：
     - `EIPPF_IR_WEAVER_BIN`
     - `EIPPF_VM_RUNTIME_LIB`
3. 编译器原命令保持不变（clang/gcc/cl/rustc）

常见 IDE：
- VS / VS Code CMake Tools / CLion：优先用 CMake launcher 注入  
- Xcode/Bazel/自研构建器：设置 `CC/CXX/RUSTC_WRAPPER` 或工具链包装命令

## 9. 验证是否生效

建议最小验证：

1. 构建日志中出现 `weaver_proxy.py`、`ip_weaver_ir` 调用  
2. 产物可正常运行（功能一致）  
3. CTest 验证通过（含 VM/runtime 测试）  

可用命令：

```bash
ctest --test-dir build --output-on-failure
```

## 10. 常见问题排查

### 10.1 `undefined reference to eippf_generated_run_template_checked`
原因：链接阶段未注入 VM Runtime。  
处理：确认 `--vm-runtime-lib` 或 `EIPPF_VM_RUNTIME_LIB` 已设置，且路径正确。

### 10.2 Rust 构建触发 strict VM fail-closed
原因：命中当前未覆盖 IR 模式。  
处理：
- 临时关闭 Rust 全覆盖：`EIPPF_RUST_FULL_COVERAGE=0`
- 或按失败函数补齐 lowering/runtime 支持后再开启。

### 10.3 NDK/跨平台链接失败
原因：VM Runtime ABI 不匹配。  
处理：为每个目标平台/架构分别构建并传入对应 runtime 库。

### 10.4 IDE 点 Build 没走代理
原因：IDE 使用了独立工具链配置，未读取 CMake launcher。  
处理：在 IDE 对应 profile/toolchain 中显式设置 launcher/wrapper。

---

## 11. 推荐落地策略

1. 本地开发：默认开启代理，保证研发阶段即验证保护兼容性  
2. CI：把代理作为唯一编译入口，防止“部分目标漏保护”  
3. 发布构建：固定 `ip_weaver_ir` 和 `eippf_vm_rt` 版本，保证可复现

## 12. Windows 适用性审查与配置

审查结论：
- 方案在 Windows 适用。  
- 关键差异是包装器必须用 `.cmd/.exe`，并正确透传 `%ERRORLEVEL%`。  
- NDK、MSVC、clang-cl 都可以接入，但路径与参数风格需区分 GNU 风格和 MSVC 风格。

### 12.1 Windows 下 clang/gcc 风格命令行

```bat
py -3 C:\path\to\core\wrapper\weaver_proxy.py ^
  --ir-weaver-bin C:\path\to\build\ip_weaver_ir\ip_weaver_ir.exe ^
  --vm-runtime-lib C:\path\to\build\runtime\eippf_vm_rt.lib ^
  clang++ -O2 main.cpp -o app.exe
```

### 12.2 Windows 下 MSVC / clang-cl 风格命令行

```bat
py -3 C:\path\to\core\wrapper\weaver_proxy.py ^
  --ir-weaver-bin C:\path\to\build\ip_weaver_ir\ip_weaver_ir.exe ^
  --vm-runtime-lib C:\path\to\build\runtime\eippf_vm_rt.lib ^
  cl.exe /O2 /c main.cpp /Fo:main.obj
```

链接阶段同理可直接走代理：

```bat
py -3 C:\path\to\core\wrapper\weaver_proxy.py ^
  --ir-weaver-bin C:\path\to\build\ip_weaver_ir\ip_weaver_ir.exe ^
  --vm-runtime-lib C:\path\to\build\runtime\eippf_vm_rt.lib ^
  link.exe main.obj /OUT:app.exe
```

### 12.3 Windows 下 IDE/构建器实践建议

1. CMake 工程：优先使用本指南第 3/4 节（最稳定）。  
2. 非 CMake 工程：把“编译器可执行路径”改为 `.cmd` 代理包装器，包装器内部再调用真实 `cl.exe/clang-cl.exe`。  
3. 保证代理返回码原样透传，否则会导致 MSBuild/Ninja 误判成功。

## 13. Windows 一键命令脚本（可直接执行）

已提供可执行脚本：
- `core\wrapper\eippf_windows_oneclick.cmd`
- `core\wrapper\eippf_vcxproj_batch_attach.cmd`
- `core\wrapper\eippf_vcxproj_batch_attach.ps1`

该脚本统一封装了：
- CMake 配置注入（自动设置 `C/CXX_COMPILER_LAUNCHER` 与 `LINKER_LAUNCHER`）
- CMake 构建
- clang/clang++ 命令行代理编译
- ndk-build（通过 `NDK_CCACHE` 自动接入）
- MSBuild（`.sln/.vcxproj`）代理接入

### 13.1 脚本完整内容（可直接复制）

```bat
@echo off
setlocal ENABLEEXTENSIONS

set "SCRIPT_DIR=%~dp0"
for %%I in ("%SCRIPT_DIR%\..\..") do set "REPO_ROOT=%%~fI"

set "PROXY=%REPO_ROOT%\core\wrapper\weaver_proxy.py"
if defined EIPPF_IR_WEAVER_BIN (
  set "IR_WEAVER=%EIPPF_IR_WEAVER_BIN%"
) else (
  set "IR_WEAVER=%REPO_ROOT%\build\ip_weaver_ir\ip_weaver_ir.exe"
)
if defined EIPPF_VM_RUNTIME_LIB (
  set "VM_RUNTIME=%EIPPF_VM_RUNTIME_LIB%"
) else (
  set "VM_RUNTIME=%REPO_ROOT%\build\runtime\eippf_vm_rt.lib"
)

if not exist "%PROXY%" (
  echo [EIPPF] Missing proxy script: %PROXY%
  exit /b 2
)
if not exist "%IR_WEAVER%" (
  echo [EIPPF] Missing IR weaver: %IR_WEAVER%
  echo [EIPPF] Set EIPPF_IR_WEAVER_BIN or build target ip_weaver_ir first.
  exit /b 2
)
if not exist "%VM_RUNTIME%" (
  echo [EIPPF] Missing VM runtime library: %VM_RUNTIME%
  echo [EIPPF] Set EIPPF_VM_RUNTIME_LIB or build target eippf_vm_rt first.
  exit /b 2
)

if "%~1"=="" goto :usage
set "MODE=%~1"
shift /1

if /I "%MODE%"=="cmake-config" goto :cmake_config
if /I "%MODE%"=="cmake-build" goto :cmake_build
if /I "%MODE%"=="clang" goto :clang_mode
if /I "%MODE%"=="ndk-build" goto :ndk_mode
if /I "%MODE%"=="msbuild" goto :msbuild_mode
if /I "%MODE%"=="vcxproj-attach" goto :vcxproj_attach_mode
if /I "%MODE%"=="help" goto :usage

echo [EIPPF] Unknown mode: %MODE%
goto :usage

:cmake_config
if "%~1"=="" (
  echo [EIPPF] cmake-config requires: ^<source_dir^> ^<build_dir^> [extra cmake args...]
  exit /b 2
)
set "SRC_DIR=%~1"
shift /1
if "%~1"=="" (
  echo [EIPPF] cmake-config requires: ^<source_dir^> ^<build_dir^> [extra cmake args...]
  exit /b 2
)
set "BUILD_DIR=%~1"
shift /1

if not exist "%BUILD_DIR%" mkdir "%BUILD_DIR%"

call :to_cmake_path "%PROXY%" PROXY_CMAKE
call :to_cmake_path "%IR_WEAVER%" IR_WEAVER_CMAKE
call :to_cmake_path "%VM_RUNTIME%" VM_RUNTIME_CMAKE

set "CACHE_FILE=%BUILD_DIR%\eippf_launcher_cache.cmake"
(
  echo set(CMAKE_C_COMPILER_LAUNCHER "py;-3;%PROXY_CMAKE%;--ir-weaver-bin;%IR_WEAVER_CMAKE%;--vm-runtime-lib;%VM_RUNTIME_CMAKE%" CACHE STRING "" FORCE^)
  echo set(CMAKE_CXX_COMPILER_LAUNCHER "py;-3;%PROXY_CMAKE%;--ir-weaver-bin;%IR_WEAVER_CMAKE%;--vm-runtime-lib;%VM_RUNTIME_CMAKE%" CACHE STRING "" FORCE^)
  echo set(CMAKE_C_LINKER_LAUNCHER "py;-3;%PROXY_CMAKE%;--ir-weaver-bin;%IR_WEAVER_CMAKE%;--vm-runtime-lib;%VM_RUNTIME_CMAKE%" CACHE STRING "" FORCE^)
  echo set(CMAKE_CXX_LINKER_LAUNCHER "py;-3;%PROXY_CMAKE%;--ir-weaver-bin;%IR_WEAVER_CMAKE%;--vm-runtime-lib;%VM_RUNTIME_CMAKE%" CACHE STRING "" FORCE^)
) > "%CACHE_FILE%"

cmake -S "%SRC_DIR%" -B "%BUILD_DIR%" -G Ninja -C "%CACHE_FILE%" %*
exit /b %ERRORLEVEL%

:cmake_build
if "%~1"=="" (
  echo [EIPPF] cmake-build requires: ^<build_dir^> [extra build args...]
  exit /b 2
)
set "BUILD_DIR=%~1"
shift /1
cmake --build "%BUILD_DIR%" %*
exit /b %ERRORLEVEL%

:clang_mode
if "%~1"=="" (
  echo [EIPPF] clang mode requires: ^<real_compiler^> [compiler args...]
  echo [EIPPF] Example: eippf_windows_oneclick.cmd clang clang++ -O2 app.cpp -o app.exe
  exit /b 2
)
set "REAL_COMPILER=%~1"
shift /1
py -3 "%PROXY%" --ir-weaver-bin "%IR_WEAVER%" --vm-runtime-lib "%VM_RUNTIME%" "%REAL_COMPILER%" %*
exit /b %ERRORLEVEL%

:ndk_mode
set "NDK_BUILD_EXE=ndk-build.cmd"
if not "%~1"=="" (
  set "NDK_BUILD_EXE=%~1"
  shift /1
)
set "CCACHE_WRAPPER=%TEMP%\eippf_ndk_weaver_ccache.cmd"
(
  echo @echo off
  echo py -3 "%PROXY%" --ir-weaver-bin "%IR_WEAVER%" --vm-runtime-lib "%VM_RUNTIME%" %%%%*
  echo exit /b %%%%ERRORLEVEL%%%%
) > "%CCACHE_WRAPPER%"

"%NDK_BUILD_EXE%" NDK_CCACHE="%CCACHE_WRAPPER%" %*
exit /b %ERRORLEVEL%

:msbuild_mode
if "%~1"=="" (
  echo [EIPPF] msbuild mode requires: ^<solution_or_vcxproj^> [extra msbuild args...]
  exit /b 2
)
set "MSBUILD_TARGET=%~1"
shift /1
set "WRAPPER_DIR=%REPO_ROOT%\core\wrapper\"
set "EIPPF_IR_WEAVER_BIN=%IR_WEAVER%"
set "EIPPF_VM_RUNTIME_LIB=%VM_RUNTIME%"
msbuild "%MSBUILD_TARGET%" /m "/p:CLToolPath=%WRAPPER_DIR%" "/p:CLToolExe=eippf_cl_wrapper.cmd" "/p:LinkToolPath=%WRAPPER_DIR%" "/p:LinkToolExe=eippf_link_wrapper.cmd" %*
exit /b %ERRORLEVEL%

:vcxproj_attach_mode
set "ATTACH_ROOT=."
if not "%~1"=="" (
  set "ATTACH_ROOT=%~1"
  shift /1
)
set "ATTACH_CMD=%REPO_ROOT%\core\wrapper\eippf_vcxproj_batch_attach.cmd"
if not exist "%ATTACH_CMD%" (
  echo [EIPPF] Missing attach command: %ATTACH_CMD%
  exit /b 2
)
"%ATTACH_CMD%" "%ATTACH_ROOT%" %*
exit /b %ERRORLEVEL%

:to_cmake_path
set "TMP_PATH=%~1"
set "TMP_PATH=%TMP_PATH:\=/%"
set "%~2=%TMP_PATH%"
exit /b 0

:usage
echo.
echo EIPPF Windows One-Click Wrapper
echo Usage:
echo   %~nx0 cmake-config ^<source_dir^> ^<build_dir^> [extra cmake args...]
echo   %~nx0 cmake-build  ^<build_dir^> [extra build args...]
echo   %~nx0 clang        ^<real_compiler^> [compiler args...]
echo   %~nx0 ndk-build    [ndk-build.cmd path] [ndk-build args...]
echo   %~nx0 msbuild      ^<solution_or_vcxproj^> [extra msbuild args...]
echo   %~nx0 vcxproj-attach [root_dir] [-DryRun]
echo.
echo Env overrides:
echo   EIPPF_IR_WEAVER_BIN
echo   EIPPF_VM_RUNTIME_LIB
exit /b 1
```

### 13.2 一键使用示例

先确保已在 Windows 机器上构建出 `ip_weaver_ir.exe` 和 `eippf_vm_rt.lib`。

1. CMake 工程（推荐，适用于 VS/CLion/VS Code CMake）：

```bat
core\wrapper\eippf_windows_oneclick.cmd cmake-config . build-win -DCMAKE_BUILD_TYPE=Release
core\wrapper\eippf_windows_oneclick.cmd cmake-build build-win -j
```

2. clang 单文件编译：

```bat
core\wrapper\eippf_windows_oneclick.cmd clang clang++ -O2 .\main.cpp -o .\main.exe
```

3. NDK（ndk-build）：

```bat
core\wrapper\eippf_windows_oneclick.cmd ndk-build ndk-build.cmd NDK_PROJECT_PATH=. APP_BUILD_SCRIPT=.\jni\Android.mk
```

4. 纯 VS（`.sln/.vcxproj`，非 CMake）：

```bat
core\wrapper\eippf_windows_oneclick.cmd msbuild .\YourSolution.sln /p:Configuration=Release /p:Platform=x64
```

5. 批量给目录下所有 `.vcxproj` 注入 `.props`（自动化）：

```bat
core\wrapper\eippf_windows_oneclick.cmd vcxproj-attach . -DryRun
core\wrapper\eippf_windows_oneclick.cmd vcxproj-attach .
```

### 13.3 VS / 其他 IDE 是否可直接默认使用

- 若 IDE 走 CMake：可以直接用上面的 `cmake-config/cmake-build` 产物目录，基本可做到“默认点击编译即走代理”。  
- 若 IDE 是自定义工具链模式：把编译器路径改为该 `.cmd`（或调用该 `.cmd clang ...`），即可保持原有构建命令语义。  
- 对纯 `.vcxproj` 非 CMake 工程，建议导入属性表模板实现“点击 Build 默认走代理”（见 13.4）。

### 13.4 纯 `.vcxproj` 默认接管（一次配置，后续点 Build 即可）

已提供模板文件：
- [eippf_vcxproj_integration.props](/workspace/core/docs/eippf_vcxproj_integration.props)

已提供工具包装器：
- `core\wrapper\eippf_cl_wrapper.cmd`
- `core\wrapper\eippf_link_wrapper.cmd`

配置步骤：
1. 在 VS 中打开 `Property Manager`。  
2. 对目标配置（如 `Release|x64`）执行 `Add Existing Property Sheet...`，选择 `eippf_vcxproj_integration.props`。  
3. 修改 `.props` 中的 `EippfWrapperDir` 为你的仓库绝对路径。  
4. 若 `ip_weaver_ir.exe` 或 `eippf_vm_rt.lib` 不在默认 `repo\build` 目录，执行：

```bat
setx EIPPF_IR_WEAVER_BIN "C:\path\to\build\ip_weaver_ir\ip_weaver_ir.exe"
setx EIPPF_VM_RUNTIME_LIB "C:\path\to\build\runtime\eippf_vm_rt.lib"
```

5. 重启 VS 后直接点击 Build 即可走代理链路。  

### 13.5 纯 VS 批量改造（自动导入 `.props`）

如果你有大量历史工程，不想逐个在 VS 里手工导入属性表，直接用：

```bat
core\wrapper\eippf_vcxproj_batch_attach.cmd C:\path\to\workspace -DryRun
core\wrapper\eippf_vcxproj_batch_attach.cmd C:\path\to\workspace
```

说明：
- 脚本会递归查找所有 `.vcxproj`。  
- 若已存在 `eippf_vcxproj_integration.props` 的导入，自动跳过（幂等）。  
- 默认把 `eippf_vcxproj_integration.props` 以相对路径写入 `ImportGroup Label="PropertySheets"`。
