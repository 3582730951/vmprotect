Architecture Contract:
- Track A / iOS Strict Gate
  - 文件：
    - `core/tools/artifact_audit.py`
    - `core/tests/tools/artifact_audit_strict_test.cpp`
  - 规则：
    - 当 `resolved_target_kind == "ios_appstore"` 或 `artifact_kind == "macho"` 时，额外执行 iOS gate
    - 以下任一成立必须 strict fail:
      - `runtime_lane != "ios_safe"`
      - `backend_kind != "ios_safe_aot"`
      - `mutation_profile != "ios_macho"`
      - `signature_policy != "required_verifier"`
      - `allow_jit != false`
      - `allow_runtime_executable_pages != false`
      - `allow_persistent_plaintext != false`
      - `require_fail_closed != true`
    - Mach-O 缺少 `LC_CODE_SIGNATURE` -> `macho_code_signature_missing`
    - Mach-O 出现 `writable_executable_segment` -> `rwx_segment_detected`
    - Mach-O imports 命中 `/System/Library/PrivateFrameworks/` 或 `PrivateFrameworks` -> `private_api_detected`
- Track B / Lane Tests
  - 文件：
    - `core/tests/lane_ios_safe/CMakeLists.txt`
    - `core/tests/lane_ios_safe/ios_safe_success_test.cpp`
    - `core/tests/lane_ios_safe/ios_safe_failure_test.cpp`
    - `core/tests/lane_ios_safe/ios_safe_security_edge_test.cpp`
    - `core/tests/CMakeLists.txt`
  - 规则：
    - success：signed Mach-O + verifier + correct manifest -> pass
    - failure：signed Mach-O + no verifier -> fail with signature gate
    - security-edge：bad manifest / private framework import / missing code signature -> fail with expected gate code
- Track C / Perf
  - 文件：
    - `core/tests/perf/ios_safe_perf_smoke_test.cpp`
    - `core/tests/perf/CMakeLists.txt`
    - `core/tests/CMakeLists.txt`
  - 规则：
    - 仿照 `windows_driver_perf_smoke_test`
    - baseline=`desktop_native`
    - compared=`ios_appstore`
    - 总体预算 `<=10%`
- Track D / CI + Evidence
  - 文件：
    - `.github/workflows/core-matrix.yml`
  - 规则：
    - `ubuntu-latest`：`configure/build/ctest/perf_smoke`
    - `macos-latest`：`configure/build` + `ctest -R 'ios_safe|artifact_audit_strict_test|post_link_mutator_smoke_test'`
    - 上传：
      - `CTest` log
      - `Testing/Temporary/LastTest.log`
      - `build cache key metadata`
      - `artifact_audit` related logs
