Architecture Contract:
- Root Contract:
  - `Phase 3d` 已经把 `string_protection_pass_test` 注册回构建图与测试图；本次 blocker 不再是注册问题。
  - 当前唯一 blocker 是 `/workspace/core/tests/passes/string_protection_pass_test.cpp` 第 59-60 行对 `llvm::ConstantExpr::getInBoundsGetElementPtr(...)` 的 brace-init indices 调用触发了 LLVM 14 下的二义性重载。
  - 本次不允许通过改 CMake、改生产 pass、改其他测试来绕过该单点源码编译错误。
- 固定修法:
  - 只允许修改 `/workspace/core/tests/passes/string_protection_pass_test.cpp`。
  - 只允许在该文件做两类改动：
    - 新增 `#include <array>`
    - 在 `build_fixture(...)` 局部把二义性的 `{zero, zero}` 改成显式固定形状
  - 固定代码形状必须如下，不允许 Engineer 自行替换成其他容器或其他 API：
    1. 保留现有 `llvm::Constant* zero = llvm::ConstantInt::get(i32_ty, 0);`
    2. 在其后新增：
       - `const std::array<llvm::Constant*, 2> gep_indices{zero, zero};`
       - `const llvm::ArrayRef<llvm::Constant*> gep_index_ref(gep_indices);`
    3. 把原来的：
       - `llvm::ConstantExpr::getInBoundsGetElementPtr(..., {zero, zero});`
       替换为：
       - `llvm::ConstantExpr::getInBoundsGetElementPtr(..., gep_index_ref);`
  - 不允许修改：
    - `build_fixture(...)` 的其他语义
    - `sink` 调用
    - 测试断言
    - marker 常量
    - `run_pass(...)`
    - 任何其他函数

  - Q1 / 是否只改 `core/tests/passes/string_protection_pass_test.cpp`
    - 是。

  - Q2 / 是否需要再改 `core/tests/passes/CMakeLists.txt`

  - Q4 / 具体修法边界是什么
    - 边界已固定为：
      - 只在 `string_protection_pass_test.cpp` 增加 `<array>` 头文件

      - 只改 `build_fixture(...)` 的该一处 GEP 调用
      - 不允许任何同文件内额外重构
