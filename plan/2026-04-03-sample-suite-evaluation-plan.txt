Task Summary:

    - Windows `.exe/.dll/.sys`
    - Linux `ELF/.so/.ko`

- Scope:

  - `core/tools/sample_suite/`
  - `.github/workflows/`

- Non-Goals:

  - 不把 Android `.ko` 伪装成带真实 GKI 兼容性证明的量产模块。
  - 不做动态破解、绕过、注入类逆向，只做产物静态可见性评估与保护前后对比。
- Constraints:
  - 样例产物编译必须走 GitHub CI。
  - 保护必须调用仓库现有入口：
    - `eippf_post_link_mutator`
    - `eippf_dex_toolchain`
    - `eippf_script_guard`
  - PM 负责任务拆分、验收、证据闭环；工程 agent 只按文件级任务卡落地。
- Acceptance Criteria:
  - 存在一条新的 CI workflow，在 CI 内完成：
    - 编译样例程序集
    - 保护样例产物
    - 逆向静态评估
    - 生成 HTML + JSON 报告
    - 上传证据包
  - 对每类样例至少生成 1 个保护后 artifact 和 1 条逆向审计记录。
  - 生成 HTML 可视化报告，至少包含：
    - 样例类型矩阵
    - 原始/保护后文件信息对比
    - strings 可见性对比
    - artifact audit / mutation / bundle / launcher 结果
    - 每类样例的结论与限制
  - 证据可下载，可由本地复查。
- Forbidden Shortcuts:
  - 禁止伪造 `.sys/.ko/.dex/.Mach-O` 文件头充数。
  - 禁止只跑本地编译然后声称“CI 已编译”。
  - 禁止只给口头结论，不落 HTML 报告和原始 JSON/日志。
  - 禁止把 hosted runner 缺失的真实平台约束伪装成“已验证量产可装载”。

Architecture Contract:
- 样例程序集不放仓库根目录，统一放在 `core/tests/sample_suite/`。
- 每类 artifact 至少一个最小样例：
  - `windows_exe`
  - `windows_dll`
  - `windows_sys`
  - `linux_elf`
  - `linux_so`
  - `linux_ko`
  - `android_so`
  - `android_dex`
  - `android_ko`
  - `ios_macho`
  - `shell_script`
- 驱动/模块样例允许是“最小可编译、最小导出、最小元数据”的样例，不要求真实安装运行。
- Android `.ko` 与 Linux `.ko` 的 artifact format 相同；如 hosted CI 无 Android kernel headers，可复用同一 `.ko` 编译方法产出独立命名样例，并在报告中明确“Android lane policy evaluation”。
- 保护入口固定：
  - `post_link_mutator` 处理 `PE/ELF/Mach-O/.sys/.ko`
  - `dex_toolchain` 处理 `.dex`
  - `script_guard` 处理 `.sh`
- `protect_and_evaluate.py` 只能消费“CI 编译阶段产出的原始 artifacts”，不得回退到仓库内其他 fixture。
- `protect_and_evaluate.py` 必须为每个输入/输出写入 `sha256`，并将 hash 绑定到 JSON summary 与 HTML 报告。
- 逆向评估固定为静态视角：
  - `file`
  - `strings`
  - `readelf` / `llvm-objdump` / `objdump`
  - `artifact_audit.py`
  - manifest / trailer / bundle header 解析
- 报告以 HTML 形式落盘，并同时导出 JSON summary。
