# 2026-04-02 Phase 2B JITEnclave Desktop-Only Confinement Plan

Execution Mode:
- `real-multi-agent`

Assurance Level:
- `heavy`

Task Summary:
- 只收口 `JITEnclavePass` / plugin / pipeline gate。
- 目标：只有 `desktop_native` 允许 JITEnclave rewrite；其他 target 命中 JIT route 必须 pass/pipeline fail-closed。

Task Charter:
- Objective:
  - 在 pass/pipeline 层完成 `JITEnclave` 的 desktop-only confinement，不依赖 runtime。
- Scope:
  - `core/passes/include/passes/JITEnclavePass.hpp`
  - `core/passes/src/JITEnclavePass.cpp`
  - `core/tests/tools/CMakeLists.txt`
  - `core/tests/tools/jit_enclave_pipeline_gate_test.cpp`
- Non-Goals:
  - 不改 runtime
  - 不改 mutator
  - 不改 wrapper
  - 不改 artifact audit
- Constraints:
  - 只允许 1 个新增测试源文件
  - gate code 必须进入编译诊断
- Acceptance Criteria:
  - `desktop_native + jit route` 编译通过
  - forbidden/unknown/missing target-kind + jit route` 编译失败
  - `stderr` 首个非空行前缀为 `error: jit_route_forbidden_for_target:`
  - `forbidden + non-jit route` 编译通过
- Forbidden Shortcuts:
  - 禁止静默跳过
  - 禁止 warning 后继续
  - 禁止缺失 target-kind 默认成 desktop

Design Decisions:
- `JITEnclavePass` target 判定唯一输入：
  - `-mllvm -eippf-target-kind=<name>`
- target 缺失时：
  - 视为 `unknown`
- 仅 `desktop_native` 允许 rewrite
- 非 desktop 命中 JIT route 时：
  - `emitError("jit_route_forbidden_for_target ...")`
  - pass fail-closed
- 非 desktop 且未命中 JIT route 时：
  - 允许通过
