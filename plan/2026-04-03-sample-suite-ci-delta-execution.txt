## 已确认根因
- Windows
  - workflow 独立预构建 plugin 使用失效的 `LLVM_DIR=C:\Program Files\LLVM\lib\cmake\llvm`
  - `find_package(LLVM CONFIG)` 失败
- macOS
  - workflow 独立预构建 plugin 未把 Homebrew LLVM clang 绑定到 CMake
  - 实际使用 `/usr/bin/c++`
  - 编译报 `llvm/Passes/PassPlugin.h` not found
- Linux
  - native sample 经过 pass 改写后，链接期缺少：
    - `eippf_string_token_decode`
    - `eippf_string_token_wipe`
- 结构问题
  - workflow 与平台脚本各自维护一套 plugin 构建逻辑，导致单一事实来源破坏
