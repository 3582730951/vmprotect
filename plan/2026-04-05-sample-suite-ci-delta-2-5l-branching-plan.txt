Branch Decision Contract:
1. `success`
   - PM 行为：
     - 不改代码
     - 下载 Windows artifact
     - 固化 run/job JSON、`windows.txt`、commands sidecar、artifact hash
     - 更新执行记录并关闭 `delta-2.5k`
   - 必保留证据：
     - `gh run view 23996885207 --json status,conclusion,headSha,jobs,url`
     - `build-windows` step 5/6/7 全 success 的原始 JSON
     - `sample-suite-windows` artifact
     - `sample_suite_inputs/toolchain_reports/windows.txt`
     - `sample_suite_inputs/toolchain_reports/commands/*`
     - `Validate Windows Toolchain Report` 全日志且无 `[FAIL]`
   - 禁止：
     - 因 Windows 通过就声称整个终局目标已额外前进；只允许关闭当前 delta

2. `failure`
   - 进入条件：
     - run `conclusion=failure` 或 `build-windows` job `conclusion=failure`
   - PM 先收集失败指纹，再开新 delta，不得反过来。
   - 失败指纹固定 4 项：
     - 第一条 `[FAIL]`
     - 失败步骤名
     - 对应 sidecar 路径
     - 第一条编译器/链接器 fatal 行
   - 失败映射：
     - `F1`
       - 指纹：`cmake configure/build for pass plugin`，且属于 LLVM 组件/链接面
       - 可改文件：
         - `/workspace/core/cmake/PassPluginCompat.cmake`
         - `/workspace/core/passes/CMakeLists.txt`
     - `F2`
       - 指纹：`cmake configure/build for pass plugin`，但属于命令拼装、路径、参数问题
       - 可改文件：
         - `/workspace/core/tools/sample_suite/build_windows.ps1`
     - `F3`
       - 指纹：`helper user compile` 或 `helper kernel compile`
       - 可改文件：
         - `/workspace/core/tools/sample_suite/build_windows.ps1`
     - `F4`
       - 指纹：`windows exe link` / `windows dll link` / `windows sys link`
       - 可改文件：
         - `/workspace/core/tools/sample_suite/build_windows.ps1`
     - `F5`
       - 指纹：`Validate Windows Toolchain Report`
       - 可改文件：
         - `/workspace/.github/workflows/sample-suite-eval.yml`
         - `/workspace/core/tools/sample_suite/build_windows.ps1`
   - 必保留证据：
     - `gh run view 23996885207 --json status,conclusion,headSha,jobs,url`
     - `gh run view 23996885207 --job 69986368454 --log`
     - 失败步骤名与失败前最后一个 success 步骤
     - 若无 artifact，则在执行记录中明确“失败于 step X，artifact 未生成”
   - 禁止：
     - 在未拿到失败指纹前直接编码
     - 任意跨越 `F1~F5` 文件边界

     - 超时阈值与达到阈值的时间戳
