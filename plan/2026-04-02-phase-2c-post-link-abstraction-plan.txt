# 2026-04-02 Phase 2C Post-Link Mutator Abstraction Split Plan

Execution Mode:
- `real-multi-agent`

Assurance Level:
- `heavy`

Task Summary:
- 本阶段只做 `Phase 2C / post_link_mutator abstraction split`，目标是把当前 `core/post_link_mutator/src/main.cpp` 的单文件实现拆分为可维护模块，并保持现有行为与 gate 不变。
- 本阶段明确不做发布级 `PE/ELF/Mach-O` 深度重写，不引入新的二进制重写语义，不扩展到 `runtime/wrapper/passes`。

Task Charter:
1. Objective:
- 将 `main.cpp` 从 600+ 行单文件拆为明确职责模块，形成 `library + thin main` 结构。
- 保持 CLI、退出码、manifest 字段、mutation trailer 格式与 `HardenTarget` 集成行为完全兼容。
2. Scope:
- `core/post_link_mutator/**`
- `core/tests/post_link_mutator/**`
- `core/tests/tools/hardened_target_pipeline_test.cpp`
- 必要时 `core/post_link_mutator/CMakeLists.txt`
- 如无必要，不改 `core/CMakeLists.txt`
3. Non-Goals:
- 不做 PE section 重排、ELF program header 重写、Mach-O load command 深改。
- 不做签名流程重构，不做 `artifact_audit.py` 语义变更。
- 不进入 Phase 3/4/5 目标。
4. Constraints:
- 只允许结构化拆分与等价重构，不允许行为漂移。
- fail-closed 语义必须保持：未知工件、未知 target、target/artifact 不匹配全部失败。
- 仅在允许目录内修改，默认不碰 `runtime/wrapper/passes`。
5. Acceptance Criteria:
- `eippf_post_link_mutator` 行为保持兼容，现有 smoke/pipeline 测试通过。
- `main.cpp` 仅保留入口编排，核心逻辑下沉到独立模块。
- 新增模块级测试覆盖 success/failure/edge-security 三类断言。
6. Forbidden Shortcuts:
- 禁止只改命名不改结构。
- 禁止通过删除校验降低失败路径。
- 禁止把“已拆分”包装成“已完成发布级 deep rewrite”。

Architecture Contract (Phase 2C Only):
1. 模块拆分清单（从 `src/main.cpp` 拆出）:
- `core/post_link_mutator/include/post_link_mutator/cli_options.hpp`

- `core/post_link_mutator/include/post_link_mutator/mutator_app.hpp`

- `core/post_link_mutator/src/mutation_trailer.cpp`
- `core/post_link_mutator/src/manifest_writer.cpp`
- `core/post_link_mutator/src/mutator_app.cpp`
- `core/post_link_mutator/src/main.cpp`（薄入口）
2. 函数级边界（固定，不留设计空间）:
- `cli_options.*` 只承载 `CliOptions`、`parse_cli(...)`、`print_usage(...)`。
- `artifact_detector.*` 只承载文件魔数读取与 `detect_base_artifact_kind(...)`。
- `target_classifier.*` 只承载 target hint 解析、target 推断、target/artifact 兼容校验、manifest artifact 分类。
- `binary_io.*` 只承载目录确保、二进制读取、二进制写入。
- `mutation_trailer.*` 只承载 FNV1a、trailer 构建、`mutate_artifact(...)`。
- `manifest_writer.*` 只承载 profile 选择、JSON 转义、manifest 写入。
- `mutator_app.*` 承载原 `main` 的完整流程与退出码映射。
- `main.cpp` 只做 `return run_mutator(argc, argv, std::cout, std::cerr);`。
3. 行为不变量（必须逐项保持）:
- mutation trailer magic 仍为 `EIPPFMT1`，version 仍为 `1`。
- `mutation_status` 仍为 `mutated_with_trailer_v1`。
- 退出码保持当前语义：`2..11` 各错误路径不变。
- `target_kind_source` 规则不变：`--target-kind` 存在时 `explicit_cli`，否则 `derived_from_target_label`。
- `HardenTarget.cmake` 调用参数与顺序不改。
