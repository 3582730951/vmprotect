## 机械步骤
1. 保留 macOS validator 当前 `required` 列表，不增删 key。
2. 保留现有：
   - `if set(kv) != set(required): raise SystemExit("[FAIL] key set mismatch")`
3. 在 `command_payload_tokens` 后、`plugin_build_command` 校验前新增：
   - `extract_cmake_define_paths(text: str, define_key: str) -> list[Path]`
   - `assert_cmake_define_matches_expected(text: str, define_key: str, expected_path: Path, fail_msg: str) -> None`
4. `extract_cmake_define_paths(...)` 只提取两种 token：
   - `-D<KEY>=...`
   - `-D<KEY>:<TYPE>=...`
   其它形式不要接受。
   固定实现方式：
   - 先 `tokens = shlex.split(text, posix=True)`
   - 再只用正则 `^-D<KEY>(?::[^=]+)?=(.+)$` 匹配单个 token
   - 匹配值直接转成 `Path`
   - 允许经 shell 还原后的带空格路径值；不接受拆成多个 token 的未闭合值
5. `assert_cmake_define_matches_expected(...)` 必须按这个顺序写：
   - 提取结果数不等于 1 -> `raise SystemExit(fail_msg)`
   - 提取值不是绝对路径 -> `raise SystemExit(fail_msg)`
   - 提取值不存在 -> `raise SystemExit(fail_msg)`
   - 先试 `candidate.samefile(expected_path)`
   - 仅当 `samefile` 抛 `OSError` 或 `FileNotFoundError` 时，再试 `candidate.resolve(strict=False) == expected_path.resolve(strict=False)`
   - 禁止 `except Exception`
   - 仍不相等 -> `raise SystemExit(fail_msg)`
6. 删除当前这两条旧判断：
   - `str(Path(kv["host_compiler_path"]).resolve()) not in plugin_build_text`
   - `str((llvm_prefix / "lib" / "cmake" / "llvm").resolve()) not in plugin_build_text`
7. 用两次函数调用替换：
   - `assert_cmake_define_matches_expected(plugin_build_text, "CMAKE_CXX_COMPILER", Path(kv["host_compiler_path"]), "[FAIL] plugin_build_command missing host compiler path")`
   - `assert_cmake_define_matches_expected(plugin_build_text, "LLVM_DIR", Path(kv["llvm_dir"]), "[FAIL] sidecar missing required token: plugin_build_command")`
8. 保持现有顺序：
   - `assert_under(plugin_build_sidecar, commands_root, "plugin_build_command")`
   - `has_forbidden_linker_controls(plugin_build_text)`
   - 再做两个 define 身份校验
9. 不改其它平台 validator
10. 不改 `build_macos_ios.sh`
11. 不改 `plugin_build_log` / `plugin_link_command` 的既有逻辑
