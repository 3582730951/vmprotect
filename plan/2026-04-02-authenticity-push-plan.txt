Architecture Contract:
- `artifact_audit` 新增 `--signature-verifier=<path>`
- `strict` policy source：
  - `--strict` + required signed target 时，必须存在 `--target-kind` 或 manifest `target_kind`
  - 仅 path suffix 命中时，在 `--strict` 下固定为 `signature_policy_unresolved`
- verifier 协议：
  - verifier 路径必须是绝对路径
  - verifier 路径必须通过信任校验：
    - allowlist 路径前缀，或
    - 固定 `sha256`
  - 相对路径、临时目录路径、信任校验失败 -> `signature_verifier_untrusted`
  - 输入参数至少含：
    - `--artifact`
    - `--artifact-kind`
    - `--target-kind`
    - `--manifest`（若存在）
    - `--artifact-sha256`
  - verifier 标准输出必须是 JSON，至少含：
    - `schema_version: 1`
    - `verified: bool`
    - `reason: str`
    - `artifact_sha256: str`
  - verifier 退出码非 `0`、JSON 非法、字段缺失、字段类型错误、stdout 为空 -> `signature_verifier_failed`
  - verifier 回传 `artifact_sha256` 与 audit 计算值不一致 -> `signature_verifier_digest_mismatch`
  - verifier 调用必须有超时与输出大小上限；超时或超限 -> `signature_verifier_failed`
- `validation_mode` 扩展：
  - `not_required`
  - `optional_format_only`
  - `optional_verifier_ignored`
  - `required_missing`
  - `required_format_invalid`
  - `policy_unresolved`
  - `required_authenticity_missing`
  - `external_verifier`
- `passed` 规则：
  - 非 required：现有逻辑保持
    - 即使提供了 verifier，只要 target 非 required，verifier 失败也不得影响通过；该路径固定为 `optional_verifier_ignored`
  - required：
    - 必须 `policy_parse_ok`
    - 必须 `parse_ok`
    - 必须 `structure_present`
    - 必须 `format_valid`
    - 必须 `authenticity_verified`
- 执行顺序：
  - 先 policy 解析
  - `policy_unresolved` 时不得调用 verifier，必须直接 fail-closed
  - verifier 路径不受信时不得调用 verifier，必须直接 fail-closed
  - 只有 `required + policy_parse_ok + parse_ok + structure_present + format_valid` 时才允许调用 verifier
- `HardenTarget`：
  - 若配置了全局 `signature verifier tool`，自动传给 audit
  - 若未配置，不影响非 signed target
