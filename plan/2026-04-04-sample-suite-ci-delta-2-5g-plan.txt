## Root Cause Judgment
- 根因固定为：
  - macOS validator 当前对三处 `-isysroot` 使用：
    - `"-isysroot" in text`
    - `str(Path(...).resolve()) in text`
  - `build_macos_ios.sh` 写入 sidecar 时保留的是原始 `IOS_SDK` 文本
  - macOS runner 上 `xcrun` SDK 路径常见 symlink / canonical 差异
  - 两者可能指向同一路径目标，但字符串不同，造成假失败
- 唯一推荐修复路径固定为：
  - 只修改 `.github/workflows/sample-suite-eval.yml`
  - 在 macOS validator 中解析 `-isysroot` flag 值
  - 用路径同一性比较代替 `resolve()` 后的字符串包含

## Phase 1 / macOS isysroot Path Identity Sync
- Owner：`Agent 2`
- File：
  - `.github/workflows/sample-suite-eval.yml`
- Exact Change：
  1. 只允许修改 macOS `Validate macOS iOS Toolchain Report` 这个 job 内嵌 Python 段；不得改同一 YAML 中 Linux / Windows / Android validator 段。
  2. 在 `command_payload_tokens` 之后新增：
     - `extract_flag_value_paths(command_text: str, flag: str) -> list[Path]`
     - `assert_flag_path_matches_expected(command_text: str, command_key: str, flag: str, expected_path: Path, fail_msg: str) -> None`
  3. `extract_flag_value_paths(...)` 固定规则：
     - 先 `tokens = command_payload_tokens(command_text)`
     - 只提取：
       - `-isysroot <path>`
       - `-isysroot=<path>`
     - 不接受其它拼写
     - 匹配值直接转成 `Path`
  4. `assert_flag_path_matches_expected(...)` 的机械规则固定为：
     - 候选数量必须恰好为 1，否则 `raise SystemExit(fail_msg)`
     - 候选必须是绝对路径，否则 `raise SystemExit(fail_msg)`
     - 候选必须存在，否则 `raise SystemExit(fail_msg)`
     - 先尝试 `candidate.samefile(expected_path)` 做同一性比较
     - 仅当 `samefile` 抛出 `OSError` 或 `FileNotFoundError` 时，才 fallback 到 `candidate.resolve(strict=False) == expected_path.resolve(strict=False)`
     - 若仍不相等，`raise SystemExit(fail_msg)`
  5. 用三次函数调用替换当前三条旧逻辑：
     - `ios_user_helper_compile_command`
       - 失败文案保持：`[FAIL] ios_user_helper_compile_command missing isysroot`
     - `ios_protected_compile_command`
       - 失败文案保持：`[FAIL] ios_protected_compile_command missing isysroot`
     - `ios_link_command`
       - 失败文案保持：`[FAIL] ios_link_command missing isysroot`
  6. 保持现有：
     - `if Path(kv["ios_isysroot"]).resolve() != ios_sdk: raise ...`
     - `has_forbidden_linker_controls(...)`
     - key-set gate
     - explicit input order gate
  7. 不改其它平台 validator
  8. 不改 `build_macos_ios.sh`
- Forbidden：
  - 不新增 schema 兼容分支
  - 不删除 Delta-2.5f / Delta-2.5e 的既有 helper 和 strict gate
  - 不把比较降级成 basename 或普通子串匹配
  - 不触碰其它 workflow job 的验证逻辑
