## Root Cause Judgment
- 根因固定为：
  - macOS validator 当前仍对以下路径做 `resolve()` 后字符串包含判断：
    - `ios_user_helper_compile_command` 的 target compiler path
    - `ios_protected_compile_command` 的 host compiler path
    - `ios_protected_compile_command` 上“不应使用 target compiler / link driver”的排斥判断
    - `ios_link_command` 的 link driver path
  - sidecar 是 `build_macos_ios.sh` 通过 `join_argv` 写出的原始参数文本
  - 原始文本路径可能与 `resolve()` 结果不同，但实际指向同一文件
  - 导致一组 path-identity 假失败依次暴露
- 唯一推荐修复路径固定为：
  - 只修改 `.github/workflows/sample-suite-eval.yml`
  - 在 macOS validator 中解析 command driver 与 wrapper `--compiler` option
  - 用路径同一性比较代替 `resolve()` 后的字符串包含

## Phase 1 / macOS compiler-driver Path Identity Sync
- Owner：`Agent 2`
- File：
  - `.github/workflows/sample-suite-eval.yml`
- Exact Change：
  1. 只允许修改 macOS `Validate macOS iOS Toolchain Report` 这个 job 内嵌 Python 段；不得改同一 YAML 中 Linux / Windows / Android validator 段。
  2. 在现有 helper 区（`assert_cmake_define_matches_expected` 后）新增：
     - `paths_identical(left: Path, right: Path) -> bool`
     - `assert_command_driver_path_matches_expected(command_text: str, command_key: str, expected_path: Path, fail_msg: str) -> Path`
     - `assert_wrapper_option_path_matches_expected(command_text: str, command_key: str, option: str, expected_path: Path, fail_msg: str) -> Path`
  3. `paths_identical(...)` 固定规则：
     - 先尝试 `left.samefile(right)`
     - 仅当抛出 `OSError` 或 `FileNotFoundError` 时，才 fallback 到 `left.resolve(strict=False) == right.resolve(strict=False)`
     - 不允许 `except Exception`
  4. `assert_command_driver_path_matches_expected(...)` 固定规则：
     - 以 `command_payload_tokens(command_text)` 的第一个 token 作为 driver 路径
     - 必须绝对路径，否则 `raise SystemExit(fail_msg)`
     - 必须存在，否则 `raise SystemExit(fail_msg)`
     - 必须与 `expected_path` 满足 `paths_identical(...)`
     - 返回解析出的 driver `Path`
  5. `assert_wrapper_option_path_matches_expected(...)` 固定规则：
     - 只解析：
       - `--compiler <path>`
       - `--compiler=<path>`
     - 候选数量必须恰好为 1，否则 `raise SystemExit(fail_msg)`
     - 候选必须绝对路径，否则 `raise SystemExit(fail_msg)`
     - 候选必须存在，否则 `raise SystemExit(fail_msg)`
     - 候选必须与 `expected_path` 满足 `paths_identical(...)`
     - 返回解析出的 option `Path`
  6. 用上述 helper 替换 5 处脆弱判定，且失败文案保持原样：
     - `ios_user_helper_compile_command missing target compiler path`
     - `ios_protected_compile_command missing host compiler path`
     - `ios_protected_compile_command must not use target helper compiler`
     - `ios_protected_compile_command must not use final link driver`
     - `ios_link_command missing link driver path`
  7. `ios_protected_compile_command` 的两条“must not use ...”固定实现：
     - 先用 `assert_wrapper_option_path_matches_expected(..., "--compiler", Path(kv["host_compiler_path"]), "[FAIL] ios_protected_compile_command missing host compiler path")` 拿到解析后的 `compiler_path`
     - 再用 `paths_identical(compiler_path, Path(kv["ios_target_compiler_path"]))` 判定并保持 `[FAIL] ios_protected_compile_command must not use target helper compiler`
     - 再用 `paths_identical(compiler_path, Path(kv["ios_link_driver_path"]))` 判定并保持 `[FAIL] ios_protected_compile_command must not use final link driver`
  8. 保持现有：
     - 三处 `assert_flag_path_matches_expected(... "-isysroot" ...)`
     - `if Path(kv["ios_isysroot"]).resolve() != ios_sdk: raise ...`
     - `assert_cmake_define_matches_expected(...)`
     - `plugin_build_log` / `plugin_link_command` strict gate
     - key-set gate
     - explicit input order gate
  9. 不改其它平台 validator
  10. 不改 `build_macos_ios.sh`
- Forbidden：
  - 不新增 schema 兼容分支
  - 不删除 Delta-2.5g / Delta-2.5f / Delta-2.5e 的既有 helper 和 gate
  - 不把比较降级成 basename 或普通子串匹配
  - 不触碰其它 workflow job 的验证逻辑
