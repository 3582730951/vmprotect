- Constraints:
  - 所有调用点必须走统一 gate
  - gate code 需要稳定、可测试、可审计
  - 不允许 safe lane 先触达可执行页 API 再失败
- Acceptance Criteria:
  - `MemoryHAL::runtime_dynamic_code_allowed()` 成为唯一动态执行页/JIT 允许入口
  - `SecureIREngine::CompileResult` 带稳定 `gate_code`
  - `eippf_execute_jit_enclave(...)` 被拒绝时可通过 `eippf_runtime_last_gate_code()` 读取 `jit_route_forbidden_for_target`
  - `eippf_execute_jit_enclave(...)` 被拒绝时探针 `probe_flags == 0x1`
  - desktop `eippf_execute_jit_enclave(...)` 正向路径必须产生允许证据
  - desktop 正向回归与 safe/unknown/dex/shell 拒绝回归共同通过
- Forbidden Shortcuts:
  - 禁止在单个调用点维护自己的 target 白名单
  - 禁止被拒绝后继续进入 `VirtualAlloc/mmap/VirtualProtect/mprotect`
  - 禁止 gate code 只写日志、不落到结果字段/API

Design Decisions:
- 唯一收口点：
  - 所有“是否允许动态执行页/JIT”的判断只走 `MemoryHAL::runtime_dynamic_code_allowed()`
- `MemoryHAL::runtime_dynamic_code_allowed()` 唯一规则：
  - 读取 `configured_target_kind()`
  - 仅当 `target_kind_supports_desktop_jit(target) == true`
  - 且 `target_forbids_runtime_executable_pages(target) == false`
  - 才返回 `true`
  - 其他 target 一律 `false`
- `jit_route_forbidden_for_target` 的唯一落点：
  - C++ IR 路径：
    - `SecureIREngine::CompileResult::gate_code`
    - 枚举值：`GateCode::kJitRouteForbiddenForTarget`
  - C ABI JIT enclave 路径：
    - `extern "C" const char* eippf_runtime_last_gate_code() noexcept`
    - 返回稳定字符串：`"jit_route_forbidden_for_target"`
- `eippf_execute_jit_enclave(...)` 必须先 gate，再做任何 resolve / 分配 / 执行
- 探针 API：
  - `extern "C" void eippf_runtime_reset_jit_enclave_probe() noexcept`
  - `extern "C" std::uint32_t eippf_runtime_jit_enclave_probe_flags() noexcept`
  - 位定义固定：
    - `0x1 = gate_checked`
    - `0x2 = resolve_attempted`
    - `0x4 = exec_alloc_attempted`
  - 禁止目标执行后探针必须精确等于 `0x1`
  - desktop 允许路径执行后必须满足 `(probe_flags & 0x7) == 0x7`

- 文件 `[secure_ir_engine.hpp](/workspace/core/runtime/include/runtime/secure_ir_engine.hpp)`

- 文件 `[secure_ir_engine.cpp](/workspace/core/runtime/src/secure_ir_engine.cpp)`

    - `result.gate_code = GateCode::kJitRouteForbiddenForTarget`

    - `result.error_message_hash` 以 `"jit_route_forbidden_for_target"` 计算
  - 其他路径不引入新的 gate code
- 文件 `[EippfRuntime.cpp](/workspace/core/runtime/src/EippfRuntime.cpp)`
  - 新增 gate code 存储
  - 新增导出：
    - `extern "C" const char* eippf_runtime_last_gate_code() noexcept`
    - `extern "C" void eippf_runtime_reset_jit_enclave_probe() noexcept`
    - `extern "C" std::uint32_t eippf_runtime_jit_enclave_probe_flags() noexcept`
  - 在 `eippf_execute_jit_enclave(...)` 开头：
    - 清空 last gate code
    - 重置 probe
    - 设置 `gate_checked`
    - 先判 `MemoryHAL::runtime_dynamic_code_allowed()`

      - 设置 `"jit_route_forbidden_for_target"`
      - 立即返回
    - 进入 resolve 前设置 `resolve_attempted`
    - 进入 executable page 分配前设置 `exec_alloc_attempted`
  - 不得在 gate 失败后继续 resolve API 或申请可执行页
