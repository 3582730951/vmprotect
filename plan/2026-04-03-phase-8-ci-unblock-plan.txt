Task Charter:
- Objective:
  - 修复 `core-matrix` 的 Linux 与 macOS hosted runner 失败，达成远端工作流全绿。
- Scope:
  - `.github/workflows/core-matrix.yml`

- Non-Goals:

  - 不在本轮扩展新的平台能力或安全特性。
  - 不把 workflow 失败通过跳过测试、关闭 gate、删 target 的方式“修绿”。
- Constraints:
  - 先计划、三审通过、再实施。
  - PM 不写实现方案之外的工程逻辑，工程 agent 只执行明确任务卡。
  - 修复后必须重新跑本地针对性验证，并推送触发远端 Actions。
- Acceptance Criteria:
  - GitHub Actions `core-matrix` 的 `linux-core` job 通过。
  - GitHub Actions `core-matrix` 的 `macos-ios-safe` job 通过。
  - `core-matrix` 绿灯必须伴随 `ios_safe` 相关测试与 `artifact_audit.py --self-test` 的执行证据。
  - macOS job 必须至少构建以下 target：
    - `artifact_audit_strict_test`
    - `post_link_mutator_smoke_test`
    - `ios_safe_success_test`
    - `ios_safe_failure_test`
    - `ios_safe_security_edge_test`
    - `ios_safe_perf_smoke_test`
  - macOS job 必须至少执行以下测试：
    - `artifact_audit_strict_test`
    - `post_link_mutator_smoke_test`
    - `ios_safe_success_test`
    - `ios_safe_failure_test`
    - `ios_safe_security_edge_test`
    - `ios_safe_perf_smoke_test`
  - 本地对应路径验证通过：
    - `post_link_mutator_thin_entry_proof_test`
    - `runtime_rust_transparent_link_test`
    - `artifact_audit_strict_test`
    - `ios_safe*`
  - 不降低已有 `ctest` / `perf_smoke` / strict audit 安全门。
- Forbidden Shortcuts:
  - 禁止在 CI 中直接排除失败测试以换取绿灯。
  - 禁止把 `Permission denied` 改成 test skip。
  - 禁止继续使用硬编码 `/workspace/...` 绝对路径。
  - 禁止在 macOS job 上靠“全量 build 失败也继续测试”伪造通过。
