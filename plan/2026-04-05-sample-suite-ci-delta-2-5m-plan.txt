Root Cause Statement:
- `delta-2.5k` 已经移除了直接 `TARGET LLVM` / `PRIVATE LLVM` 单体链接，也增加了 direct item denylist。
- 但这次失败证明：
  - `diaguids.lib` 并不是从 `llvm_map_components_to_libnames(...)` 的 direct item 文本表面进入，而是从 CMake imported target graph 的更深层 `INTERFACE_LINK_LIBRARIES` / imported location 依赖中被生成器带入 Ninja。
- 因此本轮不能再停留在“检查 direct items”，必须切到：
  - direct LLVM component target -> raw import/static library materialization
  - graph audit
  - only-under-allowed-root link closure

Architecture Contract Summary:
- Windows helper 继续使用：
  - `llvm_map_components_to_libnames(eippf_pass_plugin_llvm_libs ...)`
- 但最终 `target_link_libraries(${name} PRIVATE ...)` 不再直接吃 imported target 名。
- 新 helper 必须把 direct LLVM items 物化为实际库文件路径，要求：
  - 目标必须在 allowed LLVM root 下
  - 优先读取：
    - `IMPORTED_IMPLIB_<CONFIG>`
    - 再退到 `IMPORTED_LOCATION_<CONFIG>`
    - 再退到无配置版本
- 允许的最终 link item：
  - allowed LLVM root 下的文件路径
  - 或极小范围的 Windows system lib literal
- 禁止的最终 link item：
  - 任何包含 `diaguids.lib`
  - 任何路径含 `DIA SDK` / `dia sdk`
  - 任何不在 allowed LLVM root 下的第三方绝对库路径

Exact Change Contract:
- 只改：
  - `/workspace/core/cmake/PassPluginCompat.cmake`
- 必须新增 4 个内部 helper：
  1. `_eippf_windows_normalize_path`
     - 输入任意路径字符串
     - 输出统一斜杠、lowercase 的规范化文本
  2. `_eippf_windows_is_forbidden_link_item`
     - 检查：
       - `diaguids.lib`
       - `dia sdk`
       - `c:/program files (x86)/microsoft visual studio/2019/professional/dia sdk`
  3. `_eippf_windows_imported_target_to_library_file`
     - 输入一个 imported target
     - 按 config 读取 `IMPORTED_IMPLIB_*` / `IMPORTED_LOCATION_*`
     - 找不到则 `FATAL_ERROR`
  4. `_eippf_windows_materialize_llvm_link_items`
     - 输入 `llvm_map_components_to_libnames(...)` 得到的 direct items
     - 若 item 是 target：
       - 先做 forbidden audit
       - 再转成实际库文件路径
     - 若 item 是绝对路径：
       - 必须位于 allowed LLVM root 下
     - 若 item 是 bare `.lib`：
       - 只允许明确 system lib allowlist
       - 否则 `FATAL_ERROR`
     - 输出最终的 `eippf_pass_plugin_link_files`

- Windows 分支固定改法：
  - direct items 先保留到 `eippf_pass_plugin_llvm_libs`
  - 新增 `eippf_pass_plugin_link_files`
  - `target_link_libraries(${name} PRIVATE ${eippf_pass_plugin_link_files})`
  - 不再直接把 `eippf_pass_plugin_llvm_libs` 喂给 `target_link_libraries`

- 必须保留：
  - `WINDOWS_LLVM_COMPONENTS` hard gate
  - `llvm_map_components_to_libnames(...)`
  - `add_library(${name} SHARED ...)`
  - `PREFIX ""`
  - `WINDOWS_EXPORT_ALL_SYMBOLS ON`
  - 输出目录同步
  - target-type fail-closed

System Lib Allowlist:
