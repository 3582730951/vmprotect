Architecture Contract Delta:
- 在 `2C.3` 的“魔数优先 + 输入后缀闭口”基础上，`.dex/.sh` 再增加最小内容校验，规则写死如下：
  - `.dex`
    - 仅当输入文件前 8 字节满足以下最小模式时，才允许识别为 `ArtifactKind::kDex`：
      - 字节 `0..3` 必须是 `dex\n`
      - 字节 `4..6` 必须是 ASCII 数字
      - 字节 `7` 必须是 `\0`
  - `.sh`
    - 仅当输入文件以 shebang `#!` 开头时，才允许识别为 `ArtifactKind::kShellBundle`
- 明确禁止：
  - 不能把 `.sh` 放宽为“任意文本都算 shell”
  - 不能把 `.dex` 放宽为“任意 `.dex` 后缀都算 dex”
  - 随机二进制即使改名为 `*.dex` 或 `*.sh`，若不满足上述最小内容校验，必须返回 `ArtifactKind::kUnknown`
  - 返回 `ArtifactKind::kUnknown` 后必须继续走既有 fail-closed 路径，不能降格为 warning

Decision Log Delta:
- `DL-2C3A-01`
  - 决定：`.dex` 识别必须同时满足输入后缀和最小 DEX header 模式
  - 原因：仅凭输入后缀仍可被随机二进制伪装绕过
- `DL-2C3A-02`
  - 决定：`.sh` 识别必须同时满足输入后缀和 shebang `#!`
  - 原因：不能把任意文本或随机字节视为 shell bundle
- `DL-2C3A-03`
  - 决定：`G6 / Fail-Closed Negative` 从 2 条升级为 4 条负例全覆盖
  - 原因：原 `2C.3` 只覆盖 output-suffix 伪装，未覆盖 input-suffix fake 伪装
- `DL-2C3A-04`
  - 决定：`G5 / AC-2 Stability` 的 5 次复跑命令模板写死
  - 原因：消除验证口径解释空间

PM Task Card Delta / Core repair track:
- 在 `2C.3` 的 Core task card 第 4-5 步之间插入以下强制步骤：
  1. 当 `input_path.extension()==.dex` 时，不得立即返回 `kDex`。
  2. 必须先读取输入文件前 8 字节，并精确校验：
     - `0..3 == "dex\\n"`
     - `4..6` 为 ASCII 数字
     - `7 == '\\0'`
  3. 仅当以上全部满足时，才允许返回 `ArtifactKind::kDex`；否则返回 `ArtifactKind::kUnknown`。
  4. 当 `input_path.extension()==.sh` 时，不得立即返回 `kShellBundle`。
  5. 必须先读取输入文件前 2 字节，并精确校验 `#!`。
  6. 仅当 `#!` 命中时，才允许返回 `ArtifactKind::kShellBundle`；否则返回 `ArtifactKind::kUnknown`。
  7. 不允许引入“文本可读”“脚本像 shell”“内容像 dex”之类开放性启发式判断。

PM Task Card Delta / Test repair track:
- 在 `2C.3` 的 Test task card 第 3 步后新增以下强制步骤：

     - `fake.sh`：随机字节，不以 `#!` 开头，必须失败
     - `fake.dex`：随机字节，不含合法 DEX 最小 header，必须失败

     - `input .bin + output .dex`
  3. 四条负例必须都走真实 mutator CLI 路径，不允许只测 helper。
