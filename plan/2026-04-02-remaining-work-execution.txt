Architecture Contract:
- `target_kind` 统一契约：
  - `HardenTarget` 显式传入 `TARGET_KIND`
  - `post_link_mutator` 接收显式 `--target-kind`
  - `artifact_audit` 接收显式 `--target-kind` 或 `--manifest`
  - requirement 判断优先用显式 `target_kind`，不再优先依赖文件后缀
- `MSVC parity`：
  - 允许继续保留 GNU/Clang weak anchor
  - Windows/MSVC 增加模块内 symbol lookup 等价路径
  - 不要求 runtime library 重新按 target 重编译
- `signature gate`：
  - 输出字段仍保留：
    - `required`
    - `structure_present`
    - `authenticity_verified`
    - `validation_mode`
  - `validation_mode` 必须诚实反映能力：
    - `not_required`
    - `missing`
    - `structural_consistency`
    - `external_verifier`
  - 在当前环境无外部 verifier 时，`passed` 至少要求：
    - 策略要求存在签名
    - 签名结构存在
    - 签名负载非空/边界一致
    - target/manifest/artifact 一致
- `dex/script key binding`：
  - 继续外部 provider 模式
  - 增加 provider 协议元数据，至少绑定 `key_id`
  - 增加 provider 负向测试：缺失/拒绝/格式错误/ID 不匹配
