## 机械步骤
1. 保留 macOS validator 当前 `required` 列表，不增删 key。
2. 保留现有：
   - `if set(kv) != set(required): raise SystemExit("[FAIL] key set mismatch")`
3. 在 `assert_cmake_define_matches_expected` 后新增：
   - `paths_identical(left: Path, right: Path) -> bool`
   - `assert_command_driver_path_matches_expected(command_text: str, command_key: str, expected_path: Path, fail_msg: str) -> Path`
   - `assert_wrapper_option_path_matches_expected(command_text: str, command_key: str, option: str, expected_path: Path, fail_msg: str) -> Path`
4. `paths_identical(...)` 固定顺序：
   - 先试 `left.samefile(right)`
   - 仅当抛 `OSError` 或 `FileNotFoundError` 时，再试 `left.resolve(strict=False) == right.resolve(strict=False)`
   - 禁止 `except Exception`
5. `assert_command_driver_path_matches_expected(...)` 固定规则：
   - 以 `command_payload_tokens(command_text)` 的第一个 token 作为 driver 路径
   - 必须绝对路径，否则 `raise SystemExit(fail_msg)`
   - 必须存在，否则 `raise SystemExit(fail_msg)`
   - 必须与 `expected_path` 满足 `paths_identical(...)`
   - 返回解析出的 driver `Path`
6. `assert_wrapper_option_path_matches_expected(...)` 固定规则：
   - 只解析：
     - `--compiler <path>`
     - `--compiler=<path>`
   - 候选数必须恰好为 1，否则 `raise SystemExit(fail_msg)`
   - 候选必须绝对路径，否则 `raise SystemExit(fail_msg)`
   - 候选必须存在，否则 `raise SystemExit(fail_msg)`
   - 候选必须与 `expected_path` 满足 `paths_identical(...)`
   - 返回解析出的 option `Path`
7. 删除以下 5 条旧判定：
   - `str(Path(kv["ios_target_compiler_path"]).resolve()) not in ios_helper_text`
   - `str(Path(kv["host_compiler_path"]).resolve()) not in protected_compile_text`
   - `str(Path(kv["ios_target_compiler_path"]).resolve()) in protected_compile_text`
   - `str(Path(kv["ios_link_driver_path"]).resolve()) in protected_compile_text`
   - `str(Path(kv["ios_link_driver_path"]).resolve()) not in command_text`
8. 用 helper 替换并保持原失败文案：
   - `ios_user_helper_compile_command missing target compiler path`
   - `ios_protected_compile_command missing host compiler path`
   - `ios_protected_compile_command must not use target helper compiler`
   - `ios_protected_compile_command must not use final link driver`
   - `ios_link_command missing link driver path`
9. `ios_protected_compile_command` 的两条 “must not use ...” 固定做法：
   - 先用 `assert_wrapper_option_path_matches_expected(..., "--compiler", Path(kv["host_compiler_path"]), "[FAIL] ios_protected_compile_command missing host compiler path")` 取得 `compiler_path`
   - 再用 `paths_identical(compiler_path, Path(kv["ios_target_compiler_path"]))` 保持 `[FAIL] ios_protected_compile_command must not use target helper compiler`
   - 再用 `paths_identical(compiler_path, Path(kv["ios_link_driver_path"]))` 保持 `[FAIL] ios_protected_compile_command must not use final link driver`
10. 保留：
   - 三处 `assert_flag_path_matches_expected(... "-isysroot" ...)`
   - `if Path(kv["ios_isysroot"]).resolve() != ios_sdk: raise ...`
   - `assert_cmake_define_matches_expected(...)`
   - `plugin_build_log` / `plugin_link_command` 既有 gate
11. 不改 Linux / Windows / Android validator
12. 不改 `build_macos_ios.sh`
13. 不要引入无语义噪音改动
