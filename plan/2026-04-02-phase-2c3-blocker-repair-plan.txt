Architecture Contract Summary:
- `artifact_detector.cpp` 继续保持“魔数优先”，并收紧后缀回退：
  - `PE/ELF/Mach-O` 仍先按输入魔数识别。
  - `.dex/.sh` 只允许按 `input_path` 自身后缀识别。
  - 禁止按 `output_path` 后缀识别 `.dex/.sh`。
  - 当输入魔数未知且 `input_path` 也不是 `.dex/.sh` 时，必须返回 `ArtifactKind::kUnknown`。
- `manifest_writer.cpp` 是 `signing_profile` 的唯一真相源。目标映射固定为：
  - `windows_driver` -> `windows_driver_sign_after_mutation`
  - `linux_kernel_module` / `android_kernel_module` -> `kernel_module_sign_after_mutation`
  - `ios_appstore` -> `ios_codesign_after_mutation`
  - 其他 -> `unsigned_dev_or_sign_after_mutation`
- `post_link_mutator_smoke_test.cpp` 不是可放宽回归测试；它必须与 `manifest_writer.cpp` 的真相源精确对齐。
- `post_link_mutator_hardened_pipeline_contract_test.cpp` 是 `AC-4` 主证据，不允许“字段存在即可”的宽松校验。
- `post_link_mutator_fault_isolation_contract_test.cpp` 是 `AC-5` 主证据，必须同时验证：
  - `7/8/9` fault code
  - 非 fault 路径不返回 `7/8/9`
  - `main.cpp` 薄入口路由只调用 `run_mutator(argc, argv, std::cout, std::cerr)`
