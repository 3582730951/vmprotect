## Root Cause Judgment
- 根因固定为：
  - macOS validator 当前在 `plugin_build_command` 校验段使用：
    - `str(Path(kv["host_compiler_path"]).resolve()) not in plugin_build_text`
    - `str((llvm_prefix / "lib" / "cmake" / "llvm").resolve()) not in plugin_build_text`
  - `build_macos_ios.sh` 写入 `plugin_build_command` sidecar 时保留的是 Homebrew `opt` 路径文本
  - 在 macOS runner 上，`resolve()` 可能把 `opt` 展开到 `Cellar`
  - 两者可指向同一文件，但字符串不同，造成假失败
- 唯一推荐修复路径固定为：
  - 只修改 `.github/workflows/sample-suite-eval.yml`
  - 在 macOS validator 中解析 `plugin_build_command` 的 `-D...` define
  - 用路径同一性比较代替 `resolve()` 后的字符串包含

## Phase 1 / macOS plugin_build_command Path Identity Sync
- Owner：`Agent 2`
- File：
  - `.github/workflows/sample-suite-eval.yml`
- Exact Change：
  1. 只允许修改 macOS `Validate macOS iOS Toolchain Report` 这个 job 内嵌 Python 段；不得改同一 YAML 中 Linux / Windows / Android validator 段。
  2. 在 `command_payload_tokens` 之后、`plugin_build_command` 校验之前新增两个小函数：
     - `extract_cmake_define_paths(text: str, define_key: str) -> list[Path]`
     - `assert_cmake_define_matches_expected(text: str, define_key: str, expected_path: Path, fail_msg: str) -> None`
  3. `extract_cmake_define_paths(...)` 只允许提取：
     - `-D<KEY>=...`
     - `-D<KEY>:<TYPE>=...`
     并把值转成 `Path`
     - 固定先对 `text` 使用 `shlex.split(text, posix=True)`
     - 固定只匹配正则：`^-D<KEY>(?::[^=]+)?=(.+)$`
     - 允许经 shell 还原后的带空格路径值；不接受拆成多个 token 的未闭合值
  4. `assert_cmake_define_matches_expected(...)` 的机械规则固定为：
     - 提取结果数必须恰好为 1，否则 `raise SystemExit(fail_msg)`
     - 提取值必须是绝对路径，否则 `raise SystemExit(fail_msg)`
     - 提取值必须存在，否则 `raise SystemExit(fail_msg)`
     - 先尝试 `candidate.samefile(expected_path)` 做同一性比较
     - 仅当 `samefile` 抛出 `OSError` 或 `FileNotFoundError` 时，才 fallback 到 `candidate.resolve(strict=False) == expected_path.resolve(strict=False)`
     - 若仍不相等，`raise SystemExit(fail_msg)`
  5. 用两次函数调用替换当前 `plugin_build_command` 的两条字符串包含判断：
     - `CMAKE_CXX_COMPILER` 对比 `Path(kv["host_compiler_path"])`
       - 失败文案保持：`[FAIL] plugin_build_command missing host compiler path`
     - `LLVM_DIR` 对比 `Path(kv["llvm_dir"])`
       - 失败文案保持：`[FAIL] sidecar missing required token: plugin_build_command`
  6. 保持现有顺序不变：
     - 先 `assert_under(plugin_build_sidecar, commands_root, "plugin_build_command")`
     - 再 `has_forbidden_linker_controls(plugin_build_text)`
     - 再做 `CMAKE_CXX_COMPILER` / `LLVM_DIR` 身份校验
  7. 不改其它平台 validator
  8. 不改 `build_macos_ios.sh`
- Forbidden：
  - 不新增 schema 兼容分支
  - 不删除 Delta-2.5e 的 `plugin_build_log` / `plugin_link_command` gate
  - 不把比较降级成 basename 或普通子串匹配
  - 不触碰其它 workflow job 的验证逻辑
