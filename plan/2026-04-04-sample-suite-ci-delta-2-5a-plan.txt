## Root Cause Judgment
- 根因固定为：
  - GitHub macOS runner 使用 `/bin/bash`
  - 默认 Bash 3.2 不支持 `mapfile`
  - 当前脚本在 `build_macos_ios.sh` 的 `mapfile -t plugin_link_lines < <(...)` 处触发 `exit 127`
- 唯一推荐修复路径固定为：
  - 用 `mktemp + awk + while IFS= read -r line; do ...; done` 的 Bash 3 兼容模式替换 `mapfile`
  - 保留同一份 `PLUGIN_BUILD_LOG_PATH`
  - 保留相同的 `awk` 过滤条件
  - 保留相同的 “恰好 1 行才继续，否则 fail” 语义
  - 保留相同的 `-Wl,-dead_strip` 检查和 sidecar 写入语义

## Phase 1 / macOS Bash-3 Compatibility Closure
- Owner：`Agent 2`
- File：
  - `core/tools/sample_suite/build_macos_ios.sh`
- Exact Change：
  1. 仅替换 `mapfile -t plugin_link_lines < <( awk ... )` 这一段
  2. 新建临时文件：
     - `plugin_link_candidates_path="$(mktemp)"`
  3. 仍用现有 `awk` 过滤条件把候选 link line 写入临时文件
  4. `plugin_link_lines=()` 后，使用：
     - `while IFS= read -r line; do ...; done < "${plugin_link_candidates_path}"`
  5. 只把非空行加入数组
  6. 读完后删除临时文件
  7. 若数组长度不是 `1`，继续沿用现有 `expected exactly one...` fail 路径
  8. 只有数组长度恰好为 `1` 时，才写 `PLUGIN_LINK_COMMAND_PATH`
- Forbidden：
  - 不改 `plugin_build_cmd`
  - 不改 `PIPESTATUS[0]`
  - 不改 `PLUGIN_BUILD_LOG_PATH` / `PLUGIN_LINK_COMMAND_PATH` contract
  - 不改 `ios_protected_compile_command`
  - 不在脚本里注入 `LLVM_NO_DEAD_STRIP`
