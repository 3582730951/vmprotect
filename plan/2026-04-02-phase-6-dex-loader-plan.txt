## Threat Model
### Assets
- 加密 dex container。
- external key provider response。
- loader manifest。
- tokenized bridge table。
- 解密后的短生命周期 dex 内存缓冲区。
- strict audit 报告与 no-persistent-plaintext evidence。

### Trust Boundaries
- `bundler -> bundle/manifest` 文件边界。
- `loader -> external key provider` 端点边界。
- `loader -> Android/runtime host` 调用边界。
- `loader -> anti-analysis signals` 判定边界。
- `artifact_audit -> manifest/bundle bytes` 审计边界。

### Attacker Capabilities
- 静态分析者可离线读取产物并使用 `JADX`、`apktool`、`baksmali`、`strings`、`grep`、自定义 string-pool 扫描器。
- 动态分析者可尝试通过 `Frida`、`Xposed`、`JDWP`、ClassLoader 观察、内存 dump、fake provider、stdout/stderr 截获获取 dex 或 key。
- 攻击者不能修改本仓库冻结的 runtime contracts，但可以喂入畸形 bundle、错误 manifest、错误 key、错误 provider 端点。

### Abuse Paths
- T1 / `JADX`、`apktool`、`baksmali`、string pool grep 直接恢复业务类名与方法名
  - Likelihood: 高
  - Impact: 高
  - Mitigation:
    - release lane 只交付 `dex_bundle`，不交付 raw `.dex`。
    - bundle 内完整 dex 必须加密；bundle header 和 manifest 不允许出现类名、方法名、签名明文。
    - bridge metadata 只允许密文存储；如需 cleartext mirror，只能是 opaque token。
- T2 / `Frida`、`Xposed`、`JDWP` 在 loader 解密窗口截获 dex 或 hook bridge
  - Likelihood: 高
  - Impact: 高
  - Mitigation:
    - loader 必须有 `anti_debug_policy` 与 `anti_hook_policy` 固定字段。
    - guardrail 失败直接退出，不能继续解密。
    - 解密缓冲区短生命周期、立即清零、不得复制到 argv/env/stdout/stderr/temp file。
    - bridge 只暴露 allowlist token，不暴露原始 class/method name。
- T3 / `ClassLoader dump`、`dex dump` 通过可见 loader handle 或 sidecar 文件还原 dex
  - Likelihood: 高
  - Impact: 高
  - Mitigation:
    - manifest 固定 `class_loader_policy=private_handle_only`、`class_loader_exported=false`。
    - loader 不提供“导出 dex”“打印 class list”“列出 method list”类 CLI。
    - loader 不得创建 `.dex/.jar/.tmp/.plain` sidecar。
    - 成功路径只输出 token、计数和 gate 状态。
- T4 / external key provider 退化成静态磁盘 key file
  - Likelihood: 高
  - Impact: 高
  - Mitigation:
    - provider 端点只允许：
      - `executable_adapter`
      - `fifo`
      - `unix_socket`
    - 普通文件 provider、解析后落到普通文件的 symlink provider、工作目录静态 key file、temp dir 静态 key file 一律 `fail-closed`。
    - bundle/manifest/report 中不得回显 key material。
- T5 / manifest、report、测试日志泄露明文类名、方法名或 dex anchor
  - Likelihood: 中
  - Impact: 高
  - Mitigation:
    - manifest/report 字段固定白名单。
    - tests 必须断言 stdout/stderr/report 不出现 anchor 与类名。
    - `artifact_audit` 新增 dex lane plaintext leak gate。

## Architecture Contract
### 1. Lane 与兼容性冻结
- release lane 固定：
  - `target_kind = "android_dex"`
  - `backend_kind = "dex_loader_vm"`
  - `runtime_lane = "dex_loader_vm"`
  - `mutation_profile = "dex_bundle"`
  - `signature_policy = "required_verifier"`
  - `artifact_kind = "dex_bundle"`
  - `allow_jit = false`
  - `allow_runtime_executable_pages = false`
  - `allow_persistent_plaintext = false`
  - `require_fail_closed = true`
  - `plaintext_ttl_ms = 0`
- 兼容规则：
  - `core/runtime/include/runtime/backend_policy.hpp` 与 `core/runtime/include/runtime/backends/backend_registry.hpp` 在 Phase 6 中默认为只读兼容锚点。
  - runtime 现有 `kDexLoaderVm`、`RuntimeLaneKind::kDexLoaderVm`、`android_dex` 语义不得改名、不得降级。
  - 当前 skeleton 仍输出 `android_dex_research`；Phase 6 必须迁移为 `android_dex`。
  - 仅允许 loader/audit 在迁移窗口读取 legacy `android_dex_research` 旧 fixture；release manifest、success test、freeze 后产物一律使用 `android_dex`。

### 2. 交付物形态
- 产物 1：
  - `eippf_dex_toolchain`
  - 职责：读取 raw `.dex`，校验输入、读取 external key provider、输出 `dex bundle v3 + manifest`。
- 产物 2：
  - `eippf_dex_loader`
  - 职责：读取 `bundle + manifest + external key provider`，执行 anti-analysis preflight，内存解密，构建 tokenized bridge table，完成 loader/VM consumption。
- 产物 3：
  - `artifact_audit --strict`
  - 职责：验证 dex lane manifest contract、bundle invariant、plaintext leak、provider gate、loader gate。

### 3. `dex bundle v3` 固定格式
- magic:
  - `EDXB`
- format_version:
  - `3`
- key_material_marker:
  - 固定 `0`
- flags:
  - bit0 `bridge_table_present`
  - bit1 `guardrail_required`
- dex_version_ascii:
  - 固定 3 位 ASCII 版本号
- key_id_len
- bridge_record_count
- payload_len
- key_id bytes
- encrypted bridge table bytes
- encrypted dex payload bytes
- 禁止：
  - 在 header 区写入 class name、method name、signature、plaintext dex hash、raw key material。

### 3A. Bridge Record Contract 固定格式
- `bridge_record_count` 表示密文 bridge table 中的记录数；值必须与解密后记录条数精确一致。
- 单条 bridge record 解密后固定字段顺序为：
  - `token_u64_le`
  - `dispatch_kind_u8`
  - `method_ordinal_u32_le`
  - `flags_u16_le`
  - `opaque_metadata_len_u16_le`
  - `opaque_metadata_bytes`
- 固定值与约束：
  - `dispatch_kind_u8` 当前 phase 固定为 `1`，表示 `native_bridge_dispatch`
  - `flags_u16_le` 当前 phase 固定为 `0`
  - `opaque_metadata_bytes` 允许承载最小 bridge dispatch 所需 opaque 数据，但禁止包含 class name、method name、descriptor 明文
  - record 不允许可变前缀、字符串 key、JSON、明文 method surface
- bundler 侧若解密后任一 record 字段越界、长度不合法、或记录数与 `bridge_record_count` 不匹配，必须 fail-closed 且不得产出 bundle/manifest。

### 3B. Bridge Token Contract 固定规则
- CLI `--bridge-token=<hex_u64>` 固定为 16 个小写十六进制字符，不带 `0x`。
- bundler 生成 token 的固定输入为：
  - `key_id`
  - `method_ordinal`
  - `record_index`
  - `bridge_salt = "eippf.dex.bridge.v1"`
- 生成规则固定为：
  - 对以上 4 段按 `key_id + "\x1f" + decimal(method_ordinal) + "\x1f" + decimal(record_index) + "\x1f" + bridge_salt` 拼接
  - 计算 `fnv1a64`
  - 结果按 64 位无符号整数存储并以 16 位小写十六进制对外表示
- 唯一性规则：
  - 单个 bundle 内 token 必须唯一
  - 若生成冲突，bundler 必须 fail-closed，拒绝产物，不允许自动重试或静默覆盖
- loader 对 `--bridge-token` 的固定语义：
  - 仅检查 token 是否存在于 allowlist bridge table
  - 不打印映射名称
  - 找不到时直接非零退出

### 3C. Loader Report Contract 固定字段
- `--report=<path>` 可选；若提供，report 固定只允许写出以下白名单字段：
  - `schema_version`
  - `kind`
  - `target_kind`
  - `backend_kind`
  - `runtime_lane`
  - `loader_format_version`
  - `bridge_record_count`
  - `bridge_token_checked`
  - `bridge_token_status`
  - `guardrail_status`
  - `provider_endpoint_kind`
  - `result`
- `bridge_token_status` 固定枚举：
  - `not_requested`
  - `matched`
  - `missing`
- `guardrail_status` 固定枚举：
  - `clear`
  - `blocked_jdwp`
  - `blocked_frida`
  - `blocked_xposed`
  - `blocked_classloader_dump`
  - `blocked_dex_dump`
- report 禁止写出：
  - class name
  - method name
  - descriptor
  - dex bytes
  - key bytes
  - 明文 bridge metadata

- `schema_version: 2`
- `kind: "android_dex_bundle"`
- `target_kind: "android_dex"`
- `backend_kind: "dex_loader_vm"`
- `runtime_lane: "dex_loader_vm"`
- `mutation_profile: "dex_bundle"`
- `signature_policy: "required_verifier"`
- `artifact_kind: "dex_bundle"`
- `allow_jit: false`
- `allow_runtime_executable_pages: false`
- `allow_persistent_plaintext: false`
- `require_fail_closed: true`
- `plaintext_ttl_ms: 0`
- `loader_format_version: 3`
- `key_provider_protocol: "eippf.external_key.v1"`
- `key_provider_endpoint_kind: "executable_adapter" | "fifo" | "unix_socket"`
- `key_provider_static_file: false`
- `external_key_required: true`
- `key_id: <non-empty>`

### 5. Loader/VM 消费链
- 输入 CLI 固定为：
  - `--input-bundle=<path>`
  - `--manifest=<path>`
  - `--key-provider=<path>`
  - `--key-id=<id>`
  - `--bridge-token=<hex_u64>` 可选
  - `--report=<path>` 可选
- 固定执行顺序：
  1. 读取 manifest。
  2. 读取 bundle header。
  3. 校验 manifest 与 header 的版本、`target_kind`、`backend_kind`、`runtime_lane`、`key_id`。
  4. 解析并验证 external key provider 端点。
  5. 执行 guardrail preflight。
  6. 在内存中解密 bridge table 与 dex payload。
  7. 校验 dex magic/version。
  8. 解析最小必需 dex tables，仅构建 tokenized bridge table，不输出 class/method 明文。
  9. 若给定 `--bridge-token`，仅验证 token 是否存在并模拟 bridge dispatch 成功；不得打印映射名称。
  10. 写出白名单字段 report，清零明文缓冲区，退出。
- fail-closed 条件：
  - manifest 缺字段或字段错值。
  - bundle 不是 `EDXB v3`。
  - provider 不可读、协议坏、拒绝、key_id 不匹配、静态文件退化。
  - guardrail preflight 发现 `JDWP`、`Frida`、`Xposed`、`ClassLoader dump`、`dex dump` 风险。
  - bridge table 缺失、bridge token 不存在、dex 头无效、任何清零或 I/O 失败。

### 6. External Key Provider Contract
- `--key-provider=<path>` 只允许代表以下 3 类 provider 端点：
  - `executable_adapter`
  - `fifo`
  - `unix_socket`
- 必须沿用 shell lane 已冻结的协议名：
  - `eippf.external_key.v1`
- 明确禁止：
  - 普通文件形式的静态 key file。
  - 解析后指向普通文件的 symlink provider。
  - 工作目录中的明文 key file。
  - temp dir 中的明文 key file。
  - bundle、manifest、report、stdout、stderr、env 中出现原始 key material。
- 设计约束：
  - `core/dex_toolchain` 允许复制 Phase 5 的 provider endpoint 规则，但不允许回头改 `core/script_guard/*` 做大规模公共化重构。
  - `main.cpp` 原有“读普通文本 provider 文件”的路径必须删除，不得保留为 fallback。

### 7. Anti-Analysis Baseline
- 本仓库 Phase 6 的 on-host 交付不直接接入 Android APK 壳层，但必须把 Android threat model 编码为 loader gate。
- guardrail probe 固定覆盖：
  - `JDWP`
  - `Frida`
  - `Xposed`
  - `ClassLoader dump`
  - `dex dump`
- host 测试允许通过固定环境变量模拟上述信号：
  - `EIPPF_SIMULATE_JDWP=1`
  - `EIPPF_SIMULATE_FRIDA=1`
  - `EIPPF_SIMULATE_XPOSED=1`
  - `EIPPF_SIMULATE_CLASSLOADER_DUMP=1`
  - `EIPPF_SIMULATE_DEX_DUMP=1`
- 任一信号触发时 loader 必须非零退出，并且不得产生 report success、不得留下明文副产物。
