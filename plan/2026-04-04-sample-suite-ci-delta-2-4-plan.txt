## Exact Changes
### Track A
- 在 `build_windows.ps1` 中新增固定常量：
  - URL：
    - `https://github.com/llvm/llvm-project/releases/download/llvmorg-18.1.8/clang%2Bllvm-18.1.8-x86_64-pc-windows-msvc.tar.xz`
  - SHA256：
    - `22c5907db053026cc2a8ff96d21c0f642a90d24d66c23c6d28ee7b1d572b82e8`
  - 固定 root：
    - `C:\eippf\llvm18`
  - 固定下载路径：
    - `${env:RUNNER_TEMP}\clang+llvm-18.1.8-x86_64-pc-windows-msvc.tar.xz`
  - 固定解包 staging：
    - `${env:RUNNER_TEMP}\llvm18_unpack`
- 新增 `Ensure-PinnedLlvm18Toolchain`：
  - 若以下 3 个文件都存在则直接复用：
    - `${root}\bin\clang-cl.exe`
    - `${root}\lib\cmake\llvm\LLVMConfig.cmake`
    - `${root}\include\llvm\Passes\PassPlugin.h`
  - 否则：
    - 删除旧 archive/staging/root
    - 用 `Invoke-WebRequest` 下载 tar.xz 到固定下载路径
    - 用 `Get-FileHash -Algorithm SHA256` 校验固定 SHA
    - 若 `tar.exe` 缺失则直接 `throw`
    - 用 `tar.exe -xf <archive> -C <staging>` 解包
    - 要求 staging 下恰有一个 `clang+llvm-*` 顶层目录
    - 把该目录内容扁平复制到固定 root
    - 再次校验 3 个关键文件
- `LLVM_DIR` 固定为 `${root}\lib\cmake\llvm`
- report 中 `llvm_source` 固定写：
  - `pinned_llvm18_tarball`

### Track B
- 仅修改 `build_macos_ios.sh` 的 `ios_protected_compile_cmd`
  - 把 `-O2` 改成 `-O0`
- 其余必须保持不变：
  - `python3 ${WRAPPER}`
  - `--pass-plugin ${PASS_PLUGIN_PATH}`
  - `--compiler ${HOST_CLANGXX}`
  - `-target arm64-apple-ios17.0-simulator`
  - `-isysroot ${IOS_SDK}`
  - `-x c`
  - `-c`
  - `-o ${IOS_PROTECTED_O}`
- final link 阶段继续保持 object-only
- 这是 sample-suite/CI 特化，不是生产链路策略修改

### Track C
- Windows workflow：
  - 移除 Chocolatey 安装步骤与其对应的 `LLVMConfig.cmake` 结构假设
  - 改为只导出 deterministic tarball contract 环境变量：
    - `EIPPF_WINDOWS_LLVM_SOURCE=pinned_llvm18_tarball`
    - `EIPPF_ALLOWED_LLVM_ROOT=C:\eippf\llvm18`
    - `LLVM_DIR=C:\eippf\llvm18\lib\cmake\llvm`
  - validator 中新增 `pinned_llvm18_tarball` 分支
  - `pinned_llvm18_tarball` 分支必须绑定到固定 root
- Linux/Android workflow：
  - `.ko` token gate 的以下检查全部改成 token 语义，不再用 raw string `if token not in linux_ko_command`
    - `-nostdlib`
    - `-no-pie`
    - `-fuse-ld=lld`
    - `-Wl,-r`
  - 继续保留：
    - `has_explicit_pie_controls`
    - no-pie leakage gate
    - Android 双输入 link gate
- macOS/iOS workflow：
  - `ios_protected_compile_command` 新增 CI 特化约束：
    - 必须含 `-O0`
    - 禁止 `-O1`
    - 禁止 `-O2`
    - 禁止 `-O3`
    - 禁止 `-Ofast`
  - 保留：
    - compile-only 邻接状态机
    - pass plugin 必须存在
    - final link 不得含 plugin/source/wrapper
