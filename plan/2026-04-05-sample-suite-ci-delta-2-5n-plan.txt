Root Cause Statement:
- `delta-2.5k` 只对 `llvm_map_components_to_libnames(...)` 的 direct items 做 denylist。
- 当前失败证明：
  - `diaguids.lib` 不是 direct item 文本命中，而是从 imported target graph 的更深层 `INTERFACE_LINK_LIBRARIES` / `IMPORTED_LINK_INTERFACE_LIBRARIES_*` 中被生成器传播到 Ninja。
- 因此本轮 fix 必须：
  - 保留 direct imported targets
  - 递归遍历它们的传递依赖图
  - 定位并剪除 forbidden DIA edges
  - 再次审计 closure，确保 forbidden edge 不再存在

Architecture Contract Summary:
- Windows helper 继续使用：
  - `llvm_map_components_to_libnames(eippf_pass_plugin_llvm_libs ...)`
- 最终 `target_link_libraries(${name} PRIVATE ...)` 继续使用 imported targets：
  - `${eippf_pass_plugin_llvm_libs}`
- 新增的职责不是“替换链接模型”，而是：
  - graph audit
  - graph sanitization
  - post-sanitization verification
- 只清理 forbidden DIA-related edges，不碰正常 LLVM target graph 其余语义。

Exact Change Contract:
- 只改：
  - `/workspace/core/cmake/PassPluginCompat.cmake`
- 必须新增 5 个内部 helper：
  1. `_eippf_windows_normalize_string`
     - 统一斜杠并 lowercase，供字符串匹配
  2. `_eippf_windows_is_forbidden_dia_item`
     - 对任意 link item / path / target name 判断是否命中：
       - `diaguids.lib`
       - `dia sdk`
       - `debuginfopdb`
       - `llvmdebuginfo`
  3. `_eippf_windows_collect_target_graph`
     - 从 direct imported targets 递归收集可达 target graph
     - 只追踪 target 名与 link-interface 列表，不改链接模型
  4. `_eippf_windows_filter_forbidden_interface_list`
     - 输入一个 property list
     - 删除命中 forbidden DIA 条目的元素
     - 输出 filtered list 与 removed list
  5. `_eippf_windows_sanitize_target_graph`
     - 对 graph 中每个 imported target 处理以下 property：
       - `INTERFACE_LINK_LIBRARIES`
       - `IMPORTED_LINK_INTERFACE_LIBRARIES`
       - `IMPORTED_LINK_INTERFACE_LIBRARIES_RELEASE`
       - `IMPORTED_LINK_INTERFACE_LIBRARIES_RELWITHDEBINFO`
       - `IMPORTED_LINK_INTERFACE_LIBRARIES_MINSIZEREL`
       - `IMPORTED_LINK_INTERFACE_LIBRARIES_DEBUG`
     - 对每个 property：
       - 调用 filter helper
     - 若删掉 forbidden item，则 `set_property(TARGET ... PROPERTY ... <filtered>)`
      - 记录被删条目
     - 若未删到任何条目且最终 closure 仍含 forbidden item，则 `FATAL_ERROR`

- 遍历算法固定为：
  - `BFS`
  - 根集合固定为 `llvm_map_components_to_libnames(...)` 结果
  - 维护：
    - `queue`
    - `visited`
  - 出队前先去重
  - `visited` 命中即跳过
  - 单轮最大访问节点数固定为 `2048`
  - 超过上限立即 `FATAL_ERROR`

- ALIAS / 非 imported / 缺失 property 处理固定为：
  - `ALIAS` target：
    - 先解析 `ALIASED_TARGET`
    - 解析后的真实 target 名作为去重键
  - 非 imported target：
    - 不改 property
    - 只读取其接口并继续 forbidden 扫描
  - property 缺失、`NOTFOUND` 或空值：
    - 直接跳过
    - 不报错
    - 不写回

- generator expression 处理固定为：
  - 不做求值
  - 不改写结构
  - 先对原始字符串做 forbidden 关键字扫描
  - 只允许从以下单参数 wrapper 中提取递归 target：
    - `$<LINK_ONLY:...>`
    - `$<BUILD_INTERFACE:...>`
    - `$<INSTALL_INTERFACE:...>`
  - 其他 genex：
    - 一律不展开
    - 只保留原始文本 forbidden 扫描结果

- property 读写顺序固定为：
  - `INTERFACE_LINK_LIBRARIES`
  - `IMPORTED_LINK_INTERFACE_LIBRARIES`
  - `IMPORTED_LINK_INTERFACE_LIBRARIES_RELEASE`
  - `IMPORTED_LINK_INTERFACE_LIBRARIES_RELWITHDEBINFO`
  - `IMPORTED_LINK_INTERFACE_LIBRARIES_MINSIZEREL`
  - `IMPORTED_LINK_INTERFACE_LIBRARIES_DEBUG`
  - 每个 property 都执行：
    - 读取
    - 分词
    - 过滤 forbidden
    - 仅在有变化时写回
    - 记录 removed 项

- Windows 分支固定改法：
  - 保留 `llvm_map_components_to_libnames(...)`
  - direct items 先做 direct-text denylist
  - 再调用 `_eippf_windows_sanitize_target_graph("${name}" "${eippf_pass_plugin_llvm_libs}")`
  - sanitization 完成后，再递归检查 closure
  - 若 closure 仍含 forbidden DIA item，`FATAL_ERROR`
  - 最终仍执行：
    - `target_link_libraries(${name} PRIVATE ${eippf_pass_plugin_llvm_libs})`

- 必须保留：
  - `WINDOWS_LLVM_COMPONENTS` hard gate
  - `llvm_map_components_to_libnames(...)`
  - `add_library(${name} SHARED ...)`
  - `PREFIX ""`
  - `WINDOWS_EXPORT_ALL_SYMBOLS ON`
  - 输出目录同步
  - target-type fail-closed

Mechanical Implementation Sequence:
