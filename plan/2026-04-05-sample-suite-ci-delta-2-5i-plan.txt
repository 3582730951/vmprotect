## Task Summary
- Delta-2.5h 推送后，`sample-suite-eval` run `23994978083` 的 macOS `Validate macOS iOS Toolchain Report` 再次失败。
- 根因已在本地代码路径确认：
  - `assert_wrapper_option_path_matches_expected()` 错用 `command_payload_tokens(command_text)`
  - `command_payload_tokens` 会截断到 `--` 之后
  - `--compiler` 位于 wrapper 层，在 `--` 之前，被截断后候选为空
  - 导致 `ios_protected_compile_command` 的 host compiler path 校验继续假失败

## Root Cause Judgment
- 根因固定为：
  - `assert_wrapper_option_path_matches_expected()` 必须检查 wrapper 层参数
  - 但当前实现从 `command_payload_tokens(command_text)` 取 token
  - 对 wrapper 命令而言，`command_payload_tokens` 会移除 `--` 之前全部 token
  - 因而 `--compiler` 永远不可见
- 唯一推荐修复路径固定为：
  - 只改 `assert_wrapper_option_path_matches_expected()`
  - 改为先对完整命令做 `shlex.split(command_text, posix=True)`
  - 再只在 `--` 之前的 wrapper token 内解析 `--compiler`

## Phase 1 / wrapper-option Token Boundary Repair
- Owner：`Agent 2`
- File：
  - `/workspace/.github/workflows/sample-suite-eval.yml`
- Exact Change：
  1. 只允许修改 `assert_wrapper_option_path_matches_expected()` 的函数体；不得改函数名、参数列表、返回类型和调用点。
  2. 将：
     - `tokens = command_payload_tokens(command_text)`
     替换为：
     - `tokens = shlex.split(command_text, posix=True)`
  3. 在函数内部固定新增 wrapper token 切片逻辑：
     - 若存在 `--`，则 `wrapper_tokens = tokens[:tokens.index("--")]`
     - 若不存在 `--`，则 `wrapper_tokens = tokens`
  4. 后续 `--compiler` 候选提取只允许遍历 `wrapper_tokens`
  5. 支持的候选形式仍固定为：
     - `--compiler <path>`
     - `--compiler=<path>`
  6. 保持 fail-closed 语义不变：
     - 候选数量必须恰好为 `1`
     - 候选必须绝对路径
     - 候选必须存在
     - 候选必须与 `expected_path` 满足 `paths_identical(...)`
     - 任一失败直接 `raise SystemExit(fail_msg)`
  7. 保持以下失败文案完全不变：
     - `[FAIL] ios_protected_compile_command missing host compiler path`
     - `[FAIL] ios_protected_compile_command must not use target helper compiler`
     - `[FAIL] ios_protected_compile_command must not use final link driver`
  8. 不改其它 helper：
     - `command_payload_tokens`
     - `paths_identical`
     - `assert_command_driver_path_matches_expected`
     - `assert_flag_path_matches_expected`
     - `assert_cmake_define_matches_expected`
