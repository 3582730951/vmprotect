## Phase 2C1-B：退出码 2..11 与 Trailer Magic/Version 硬断言

PM Task Card 2C1-B.1（新增退出码与 trailer 合同测试）
1. 新增文件：
- `core/tests/post_link_mutator/post_link_mutator_exit_code_trailer_contract_test.cpp`
2. 测试必须覆盖并硬断言：
- `exit code = 2`：参数缺失/非法 CLI。
- `exit code = 3`：输入文件不存在。
- `exit code = 4`：未知工件魔数。
- `exit code = 5`：非法 `--target-kind` hint。
- `exit code = 6`：target/artifact 不匹配。
- `exit code = 7`：输入读取失败（通过测试钩子强制）。
- `exit code = 8`：backend 解析为 unknown（通过测试钩子强制）。
- `exit code = 9`：mutation 输出与输入相同（通过测试钩子强制）。
- `exit code = 10`：输出文件写入失败。
- `exit code = 11`：manifest 写入失败。
3. 测试钩子固定方案（不给工程 Agent 设计空间）：
- 在 `post_link_mutator` 内增加仅供测试调用的 `run_mutator_with_test_fault(...)`。
- 固定 fault 枚举：
  - `kNone`
  - `kForceReadInputFailure`（命中 code 7）
  - `kForceBackendUnknown`（命中 code 8）
  - `kForceMutationIdentity`（命中 code 9）
- `main.cpp` 仍只调用无 fault 的 `run_mutator(...)`；生产路径不可启用 fault。
4. Trailer 硬断言（同一测试内必须包含）：
- 对成功变异输出读取尾部 trailer，硬断言：
  - magic 固定为 `EIPPFMT1`。
  - version 固定为 `1`。
- 任一不符直接 FAIL。

PM Task Card 2C1-B.2（CMake 与 ctest 注册）
1. 在 `core/tests/post_link_mutator/CMakeLists.txt` 新增：
- `post_link_mutator_exit_code_trailer_contract_test` target。
- 对应 `add_test(...)` 注册。
