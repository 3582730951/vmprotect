## 机械步骤
1. 新建文件：

2. 在该文件中定义固定 helper：
   - `function(eippf_add_pass_plugin_target name)`
   - `cmake_parse_arguments(ARG "" "OUTPUT_NAME" "SOURCES;COMPILE_DEFINITIONS" ${ARGN})`
3. helper 内按以下顺序写，不允许换顺序：
   - `string(TOUPPER ${name} name_upper)`
   - `set(LLVM_${name_upper}_LINK_INTO_TOOLS OFF CACHE BOOL "Force EIPPF pass plugins to remain loadable modules." FORCE)`
   - `if(WIN32)`
   - Windows 分支里先判断：
     - `if(TARGET LLVM)` 走 `target_link_libraries(${name} PRIVATE LLVM)`
     - `elseif(COMMAND llvm_map_components_to_libnames)` 走：
       - `llvm_map_components_to_libnames(eippf_pass_plugin_llvm_libs Core Passes Support Analysis TransformUtils ScalarOpts InstCombine ipo Instrumentation IRReader)`
       - `target_link_libraries(${name} PRIVATE ${eippf_pass_plugin_llvm_libs})`
     - `else() message(FATAL_ERROR ...) endif()`
   - Windows 分支创建 target：
     - `add_library(${name} SHARED ${ARG_SOURCES})`
     - `set_target_properties(${name} PROPERTIES PREFIX "" WINDOWS_EXPORT_ALL_SYMBOLS ON)`
     - 若 `CMAKE_LIBRARY_OUTPUT_DIRECTORY` 非空，同步到：
       - `RUNTIME_OUTPUT_DIRECTORY`
       - `LIBRARY_OUTPUT_DIRECTORY`
     - 若 `CMAKE_LIBRARY_OUTPUT_DIRECTORY_RELEASE` 非空，同步到：
       - `RUNTIME_OUTPUT_DIRECTORY_RELEASE`
       - `LIBRARY_OUTPUT_DIRECTORY_RELEASE`
   - `else()` 非 Windows 分支：
     - `add_llvm_pass_plugin(${name} ${ARG_SOURCES})`
   - `endif()`
   - 读取 target type：
     - `get_target_property(eippf_target_type ${name} TYPE)`
   - 若 target type 不是 `MODULE_LIBRARY` 或 `SHARED_LIBRARY`：
     - `message(FATAL_ERROR "...non-compilable...")`
   - 统一公共属性：
     - `target_include_directories(${name} PRIVATE "${CMAKE_CURRENT_SOURCE_DIR}/include")`
     - `target_include_directories(${name} SYSTEM PRIVATE ${LLVM_INCLUDE_DIRS})`
     - `if(ARG_COMPILE_DEFINITIONS) target_compile_definitions(${name} PRIVATE ${ARG_COMPILE_DEFINITIONS}) endif()`
     - `eippf_apply_common_compile_options(${name})`
     - `if(ARG_OUTPUT_NAME) set_target_properties(${name} PROPERTIES OUTPUT_NAME "${ARG_OUTPUT_NAME}") endif()`
4. 只修改 `/workspace/core/passes/CMakeLists.txt`
5. 在 `include(AddLLVM)` 之后立刻加入：
   - `include(PassPluginCompat)`
6. 删除所有直接 `add_llvm_pass_plugin(...)` 调用。
7. 用下面 8 个固定 helper 调用替换原有块，顺序不得变化：

   - `eippf_add_pass_plugin_target(eippf_selective_vm_pass SOURCES src/SelectiveVMPass.cpp COMPILE_DEFINITIONS EIPPF_SELECTIVE_VM_STANDALONE_PLUGIN=1)`
   - `eippf_add_pass_plugin_target(eippf_mba_obfuscation_pass SOURCES src/MBAObfuscationPass.cpp COMPILE_DEFINITIONS EIPPF_MBA_OBFUSCATION_STANDALONE_PLUGIN=1)`
   - `eippf_add_pass_plugin_target(eippf_iat_minimization_pass SOURCES src/IATMinimizationPass.cpp COMPILE_DEFINITIONS EIPPF_IAT_MINIMIZATION_STANDALONE_PLUGIN=1)`
   - `eippf_add_pass_plugin_target(eippf_jit_enclave_pass SOURCES src/JITEnclavePass.cpp COMPILE_DEFINITIONS EIPPF_JIT_ENCLAVE_STANDALONE_PLUGIN=1)`

   - `eippf_add_pass_plugin_target(eippf_protection_suite_pass SOURCES src/ProtectionSuitePassPlugin.cpp src/StringProtectionPass.cpp src/ProtectionAnchorPass.cpp src/SelectiveVMPass.cpp src/JITEnclavePass.cpp src/CFFObfuscationPass.cpp OUTPUT_NAME eippf_protection_suite_pass)`
