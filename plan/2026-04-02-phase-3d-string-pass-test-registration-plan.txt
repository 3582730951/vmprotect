Architecture Contract:
- Root Contract:
  - 本 blocker 的根因只在 `core/tests/passes/CMakeLists.txt` 的测试目标注册条件与链接策略。
  - `core/tests/CMakeLists.txt` 已经把 `passes` 子目录纳入构建图；本次不允许通过修改顶层 tests 接线补洞。
  - `core/passes/CMakeLists.txt` 已经产出 `eippf_string_protection_pass` 与 `eippf_protection_suite_pass`；本次不允许通过修改生产 pass 接线补洞。
  - `core/tests/passes/string_protection_pass_test.cpp` 已存在且语义稳定；本次不允许改测试源来规避注册问题。
- 固定修法:
  - 只允许修改 `/workspace/core/tests/passes/CMakeLists.txt`。
  - 必须保留：
    - 目标名 `string_protection_pass_test`
    - 源文件列表中的 `string_protection_pass_test.cpp`
    - 源文件列表中的 `"${CMAKE_SOURCE_DIR}/passes/src/StringProtectionPass.cpp"`
    - 测试名 `string_protection_pass_test`
  - 必须移除当前“整个测试目标被 `if(TARGET LLVM)` 静默包裹”的注册方式。
  - 必须把链接策略改成仓库内已存在、已证明可用的 `ip_weaver_ir` 同款 fallback 结构：
    - 先尝试 `if(TARGET LLVM)` -> `target_link_libraries(... PRIVATE LLVM)`
    - 否则 `elseif(COMMAND llvm_map_components_to_libnames)` -> 使用该命令解析并链接下列固定组件库：
      - `Core`
      - `Support`
      - `BitReader`
      - `BitWriter`
      - `IRReader`
      - `Passes`
      - `Analysis`
      - `TransformUtils`
    - 不允许继续保留“`TARGET LLVM` 不成立就整个测试不注册”的静默跳过路径
  - LLVM include 路径继续沿用当前测试侧合同：
    - `passes/include`
    - `EIPPF_LLVM_INCLUDE_DIRS`
- 明确回答 5 个问题:
  - Q1 / 是否只改 `core/tests/passes/CMakeLists.txt`
    - 是。
    - 原因：顶层 tests 已纳入 `passes` 子目录，生产 pass targets 已存在，缺口集中在该文件内部的 guard 与 link strategy。
  - Q2 / 是否需要改 `core/tests/CMakeLists.txt`
    - 不需要。
    - 原因：`core/tests/CMakeLists.txt` 已调用 `eippf_maybe_add_test_subdir(passes)`；`build_phase3/tests/passes/CTestTestfile.cmake` 为空说明子目录已进入图，但子目录自身没有产出测试条目。
  - Q3 / 是否需要改 `core/passes/CMakeLists.txt`
    - 不需要。
    - 原因：`eippf_string_protection_pass` 与 `eippf_protection_suite_pass` 已存在且可构建，当前 blocker 不是生产 pass target 缺失。
  - Q4 / 是否允许改测试源 `core/tests/passes/string_protection_pass_test.cpp`
    - 不允许。
    - 原因：当前 evidence 只证明构建/注册缺口，没有任何测试语义或源码缺陷证据；改源码会扩大 blast radius 并掩盖根因。
  - Q5 / 为什么修法边界要写死成上面这样
    - 因为当前构建图中：
      - `LLVM_DIR` 已存在
      - `EIPPF_LLVM_INCLUDE_DIRS` 已存在
      - pass targets 已存在
      - 唯独 `TARGET LLVM` 不成立导致 `string_protection_pass_test` 整体不被定义
    - 所以最小 delta 必须只修 `tests/passes` 的 guard 与 link fallback，不改顶层 tests、不改生产 pass、不改测试源码。
