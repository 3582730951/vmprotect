## Phase 1 / macOS Validator Contract Sync
- Owner：`Agent 2`
- File：
  - `.github/workflows/sample-suite-eval.yml`
- Exact Change：
  1. 只允许修改 macOS `Validate macOS iOS Toolchain Report` 这个 job 内嵌 Python 段；不得改同一 YAML 中 Linux / Windows / Android validator 段。
  2. 在 macOS `Validate macOS iOS Toolchain Report` 的 `required` key set 中保留当前旧完整列表，并只新增：
     - `plugin_build_log`
     - `plugin_link_command`
  3. 必须保留 `if set(kv) != set(required): raise SystemExit(\"[FAIL] key set mismatch\")` 这条 fail-closed 断言。
  4. 对 `plugin_build_log` 增加独立校验，固定复用现有 `Path(value).is_absolute()` 与 `assert_under(sidecar, commands_root, label)` 风格：
     - 缺失时报 `[FAIL] missing key: plugin_build_log`
     - 非绝对路径时报 `[FAIL] non-absolute path for plugin_build_log`
     - 不存在时报 `[FAIL] missing path for plugin_build_log`
     - 越过 commands 根时报 `[FAIL] plugin_build_log escaped allowed root`
  5. 对 `plugin_link_command` 增加独立校验，固定放在 `plugin_build_command` 校验之后、`ios_user_helper_compile_command` 校验之前，且失败文案必须统一为 `[FAIL] ...`：
     - sidecar 文件存在
     - `Path(...).is_absolute()`
     - `assert_under(sidecar, commands_root, "plugin_link_command")`
     - 内容 `strip()` 后非空，否则报 `[FAIL] plugin_link_command empty`
     - 内容必须包含 `str(Path(kv["plugin_path"]).resolve())` 这个绝对路径令牌，否则报 `[FAIL] plugin_link_command missing plugin path`
     - 内容不得匹配正则 `r'^\[[0-9]+/[0-9]+\]\s+'`，否则报 `[FAIL] plugin_link_command leaked ninja progress prefix`
     - 内容不得包含 `-Wl,-dead_strip`，否则报 `[FAIL] plugin_link_command leaked dead_strip`
  6. 不改其它平台 validator
  7. 不改 `build_macos_ios.sh`
- Forbidden：
  - 不新增 schema 兼容分支
  - 不删除 Delta-2.5b 契约
  - 不触碰其它 workflow job 的验证逻辑
  - 不把 `plugin_link_command` 的 plugin 路径校验降级为仅检查 basename
