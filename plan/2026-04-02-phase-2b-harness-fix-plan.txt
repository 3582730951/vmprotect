# 2026-04-02 Phase 2B Harness Fix Plan

Execution Mode:
- `real-multi-agent`

Assurance Level:
- `heavy`

Task Summary:
- 仅修复 `Phase 2B` gate test harness 用错编译器的问题。
- 不改 pass 语义，不改 runtime。

Task Charter:
- Objective:
  - 让 `jit_enclave_pipeline_gate_test` 始终由真实 `clang/clang++` 驱动执行，消除 `g++` + `-fpass-plugin` 造成的假失败。
- Scope:
  - `core/tests/tools/CMakeLists.txt`
  - `core/tests/tools/jit_enclave_pipeline_gate_test.cpp`
- Non-Goals:
  - 不改 `JITEnclavePass`
  - 不改 runtime
  - 不减少 gate 测试矩阵
- Constraints:
  - 不能硬编码 `/usr/bin/clang`
  - 不能回退到 `g++`
- Acceptance Criteria:
  - 配置阶段能稳定解析真实 clang 路径
  - `jit_enclave_pipeline_gate_test` 实际矩阵执行通过
- Forbidden Shortcuts:
  - 禁止找不到 clang 时继续配置
  - 禁止跳过失败矩阵 case
  - 禁止通过修改 pass 逻辑掩盖 harness 问题

Design Decisions:
- `jit_enclave_pipeline_gate_test` 的编译器来源统一为独立 CMake 变量：
  - `EIPPF_TEST_CLANG_PATH`
- 解析顺序固定：
  - 若显式提供 `EIPPF_TEST_CLANG_PATH`，优先使用
  - 否则若 `CMAKE_CXX_COMPILER_ID == Clang`，使用 `${CMAKE_CXX_COMPILER}`
  - 否则 `find_program(... NAMES clang++ clang)` 自动发现
- 若最终未找到 clang 或解析结果不是 clang：
  - 配置阶段 `FATAL_ERROR`
- `jit_enclave_pipeline_gate_test.cpp` 在执行矩阵前必须自检：
  - 跑 `EIPPF_CLANG_PATH --version`
  - 输出必须包含 `clang`
  - 否则测试立即失败，错误文本包含 `toolchain_not_clang`
